- Portal description (Description): Hi Sean, Our department’s goal is to track the average days it takes to schedule patients. I’ve created this report, which tracks the days between two stages in consultation opportunities. The old value being New, or Needs Scheduling (as this is the default stage once a lab consultation opp is auto-created). And the new value is “Scheduled.” I’ve got the report in a good spot, after QA’ing several records, it appears to be accurate. What I need to add to this report is columns for the billing state and the main practitioner fields on each patient’s account. This is needed to that we can narrow down TTS on a state and provider level. Right now, I have achieved this by cross-referencing this SF opp data with appointment data in the EHR, locating the matches and analysing those schedule times in an excel doc. It would be amazing if all of this data could live in the one salesforce report, so that my Excel doc has less margin for error in the match formulas. Since this report type is opportunity field history, I can’t get account-level information onto it. There are main practitioner and state fields at the opportunity level, but they are blank because no data pulls into them from the account itself. I’ve recreated this report for genetic test clarity calls and for peptide express consultations. Ideally the same edits you make to the lab review report should be copied onto the other two product lines. Lab review report is definitely the highest priority. Happy to clarify on anything if needed. I really appreciate your help!
