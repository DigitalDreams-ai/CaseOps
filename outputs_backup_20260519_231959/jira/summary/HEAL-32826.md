- Portal description (Description): Please assign to Sean We identified several hundred PAM cases (3/18–3/27) for Shopify orders missing Informed Consent. It appears the fix released on 3/27 resolved the issue—recent MGT orders now have “Informed Consent 2026” on the SF account. Example: For affected cases, patients purchased MGTs/GHTs but have no Informed Consent recorded in SF. It seems consent assignment (likely via SF on account/opportunity creation) failed during this window. We’re considering using the “Assign patient agreements” action to backfill, but need confirmation on the intended workflow for Shopify purchases. Our understanding is: consent is assigned via Shopify → synced to SF → then to Chrono (and removed from SF). That flow does not appear to be functioning during this time period (and the removal from SF does not appear to occur currently or historically). Could @Sean Bingham investigate on the SF side and Adam on the eComm side to clarify: How Informed Consent is assigned for MGT/GHT purchases Which system is the source of truth How consent syncs across platforms Whether “Assign patient agreements” is the correct remediation This may impact hundreds of patients lacking required consents, so we consider it urgent. Should we open a bug ticket in ADO?
