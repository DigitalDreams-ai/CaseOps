I investigated this issue. It requires an Engineering change and has been escalated.

**Problem:**

When the "Generate Pharmacy Order" button is used on an Opportunity, the autolaunched flow `Opportunity_to_Order` correctly retrieves the DEA Number, State License Number, and Patient Driver's License from the Opportunity — but it never writes those values onto the new Order record. Because the Order is created with blank fields, the "Purchase Order with Scripts" PDF renders a blank DEA Number on page 1 (and the other fields are likely blank as well).

This is a regression from HEAL-30582, meaning the previous fix was overwritten at some point — likely by a deployment that used an older version of the flow.

**What needs to change:**

The `Assign Order Fields` assignment element inside the `Opportunity_to_Order` flow needs to be updated to include three missing field mappings:
- DEA Number (`DEA_Number__c`)
- State License Number (`State_License_Number__c`)
- Patient Driver's License (`Patient_Driver_s_License__c`)

Once those assignments are in place, the values will carry over from the Opportunity to the Order at creation time, and the PDF will populate correctly. The fix will need to be built and validated in Sandbox, then deployed to Production.

**Evidence:**

- Affected Order record: `801Ql00000v0QZdIAM` — confirmed blank DEA/DL fields post-creation.
- Attached PDF (`Purchase Order with Scripts - 2026-03-18T153921.225.pdf`) — blank DEA on page 1.
- Root cause confirmed via direct flow inspection by Sean Bingham (2026-03-24).

**Escalation notes:**

This has been escalated to Engineering with full reproduction steps, root cause details, and the specific flow element that needs to be corrected. We are also flagging the regression concern so Engineering can investigate why the HEAL-30582 fix was overwritten and put controls in place to prevent a third recurrence.

We will update you once the fix has been validated in Sandbox and is ready for Production deployment. In the meantime, your existing workaround remains available.
