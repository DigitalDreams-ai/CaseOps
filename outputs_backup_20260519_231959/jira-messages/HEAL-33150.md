I've completed my initial investigation of this issue.

**Problem:**
When an email is sent from a **Cx Case** record in Salesforce, the outbound email correctly includes the Salesforce case thread ID. However, when the recipient replies from an external email client (e.g., Outlook), their reply is **not syncing back into Salesforce** and is not being logged on the Case Activity timeline. This leaves the Case communication history incomplete and breaks the audit trail.

**What needs to change:**
This is a configuration issue with how the **Cx Case Record Type** is set up within Salesforce's **Email-to-Case** routing. Specifically, one or more of the following needs to be verified and corrected:

1. **Email-to-Case Routing Address** — The routing address linked to the Cx Case Record Type must have email threading enabled. If threading is off or the routing address is misconfigured, inbound replies never reach Salesforce.
2. **Reply-To address on outbound Cx Case emails** — The Reply-To header on emails sent from Cx Case records must point to the Salesforce routing/inbound address, not an agent's personal email. If it points to an Outlook inbox, replies go there and Salesforce never sees them.
3. **Email template thread token** — The email template(s) used on Cx Cases must include the Salesforce thread ID merge field so inbound replies can be matched to the correct Case.

I am currently working through these configuration points in Sandbox. Once I have confirmed the fix, I will update you with the exact change made and the testing results before promoting to Production.

**Evidence:**

**Important note on historical replies:**
Unfortunately, emails that were already sent and replied to outside of Salesforce (like Tyler's reply in your example) **cannot be automatically recovered** into the Case. Those specific gaps in the historical thread are permanent. The fix will ensure that all future replies are correctly captured going forward. If you need those historical replies documented, they would need to be manually added as a Case note or internal comment.
