# Jira Message Draft

Hi Jennifer Lara,

Thank you for the detailed report. I've completed my investigation of this issue and wanted to share my findings.

**Problem:**
The three PGT Booklet orders received by On Global were submitted to Novogenia with product code `#VRSF - Report Print & Shipment Fee` included in the original API request — Novogenia received and fulfilled these printed reports, which is consistent with what you observed. However, the Salesforce integration did not create corresponding Lab Order or Opportunity records for the PGT Booklet component of these orders, leaving them "orphaned" on the Salesforce side.

**What needs to change:**
This is a gap in the Novogenia integration layer (the code or automation that constructs and sends the API payload to Novogenia). The component that includes `#VRSF` in the outbound request either never had — or is missing — a step to create the corresponding Salesforce record. Because the original submission logs for these orders fall outside our accessible debug log window (at least two of the orders are from 2024), this requires Engineering to access extended logs or the integration codebase directly to confirm the exact failure point and implement a fix.

**Evidence (detailed SOQL audit):**
- **Novogenia Portal confirms `#VRSF`:** Courtney Marlin verified `#VRSF - Report Print & Shipment Fee` is present on all three fulfilled orders in Novogenia.
- **Salesforce Lab Orders exist but are incomplete:**
  - LAB-000228814 (a47Ql000000JI2rIAG, 03/02/2026)
  - LAB-000232068 (a47Ql000000Lo9tIAC, 04/07/2026)
  - LAB-000058193 (a47Rh0000001zeEIAQ, 05/27/2024)
  
  Each is linked to the correct Opportunity and has ONE OpportunityLineItem for the main PGT product ($1299). **None have the "Printed Report Books" product or any reference to `#VRSF`.**
- **Salesforce has no `#VRSF` product code mapping:** Query for `ProductCode LIKE '%VRSF%'` returned zero results. The code does not exist in Salesforce Product2.
- **The "Printed Report Books" product exists but is unused:** Salesforce Product2 ID `01tRh000006VKS1IAO` (the semantic equivalent of Novogenia's `#VRSF`) has no ProductCode, no Pricebook entries, and is not linked to any of the affected Opportunities.
- **No data model for supplemental products:** Lab Orders support only a single `Product__c` field (not a collection). No `Lab_Order_Product__c` junction object exists.
- **Affected Opportunities:** `006Rh000009YUMfIAO`, `006Rh00000GVH6cIAH`, `006Ql00000YpadlIAB` — each has only the main PGT product line item; no booklet record.

**Escalation notes:**
This issue has been escalated to our Salesforce Engineering team for investigation and resolution. Engineering will need to:
1. Retrieve and review the original API call logs for the three Lab Orders.
2. Identify the specific code or automation component responsible for the missing Salesforce record creation.
3. Implement a fix to prevent future orphaned PGT Booklet orders.
4. Assess whether additional orphaned orders exist beyond the three identified here.
5. Determine next steps for retroactively addressing the three known affected records.

Please note that Engineering is currently working through a queue of high and critical priority items, so there may be a delay before a resolution is delivered. We will keep this ticket updated as progress is made.

In the meantime, please continue with your existing workaround and let us know if any additional orders surface with this same pattern so we can add them to the scope of investigation.

Thanks for your patience and for the thorough documentation of this issue — it was very helpful in identifying the affected records quickly.
