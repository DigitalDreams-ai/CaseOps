**Problem:**
The **Order Notes** field on the Opportunity object has not been enabled for **Salesforce Field History Tracking**. Salesforce requires each field to be individually opted in to tracking — since "Order Notes" was never added to the tracked field list, edits to it are not recorded in the Opportunity Field History related list. This is a configuration gap, not a bug.

**What needs to happen:**
The fix is straightforward in most cases: enable Field History Tracking for the "Order Notes" field via **Setup → Object Manager → Opportunity → Set History Tracking**. Once enabled, all future edits to that field will appear in the Field History related list showing who changed it, when, and the old and new values.

Before I apply the fix, I need to confirm two things:

1. **Field type of "Order Notes"** — Salesforce's native Field History Tracking does **not** support Long Text Area or Rich Text Area fields (this is a Salesforce platform limitation). If "Order Notes" is one of these types, a different technical solution will be required, and I'll escalate accordingly. I'm checking this now.
2. **Current tracked field count** — Salesforce allows a maximum of 20 tracked fields per object. I need to confirm the Opportunity object is under this limit before adding another field.

**Important note on historical data:**
Salesforce Field History Tracking is **not retroactive**. Once enabled, only changes made *after* the tracking is turned on will be captured. Edits made to "Order Notes" before this change cannot be recovered — this is a platform limitation.
