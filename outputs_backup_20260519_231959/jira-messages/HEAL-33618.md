Thank you for submitting this request. I've completed my investigation of HEAL-33618.

**Problem:**
Salesforce's standard Case object does not record a timestamp when a Case is escalated. The standard `IsEscalated` field is a boolean (checked/unchecked) only — it does not capture *when* the escalation occurred. Without an escalation timestamp stored on the Case record, there is no data point available to calculate elapsed time between Case open → escalation or escalation → close in any Salesforce report.

**What needs to change:**
This requires an Engineering change. Specifically, our team needs to:

1. **Create a custom Date/Time field** (`Escalation Date/Time`) on the Case object to record the exact moment a Case is escalated from Tier 1 to Tier 2.
2. **Build an automated Flow** that stamps this field in real time when the escalation event occurs.
3. **Build a report** (and optionally formula fields) that surface the escalation timestamp alongside Case Created Date and Closed Date, with elapsed-time calculations for both intervals.

**Before Engineering can begin, we need one key clarification from you:**

> **How is escalation currently triggered in your workflow?** For example:
> - Does a Tier 1 agent check a checkbox or toggle a field on the Case?
> - Does the Case Status change to a specific value (e.g., "Escalated")?
> - Is the Case reassigned to a specific Tier 2 queue or agent?
> - Is there an automated Escalation Rule in Salesforce that fires?

This answer determines exactly where and how the automation is built. Once we have this, Engineering can proceed immediately.

**Additional clarification (lower priority, but helpful):**
- Do you need elapsed time based on **business hours** (excluding nights/weekends) or **calendar hours**?
- Should the system record only the **first** escalation time, or the most recent one?

- You will be asked to validate the solution in Sandbox before it is deployed to Production.
- Please note: **historical Cases escalated before this solution is deployed will not have a retroactive escalation timestamp.** Reporting will be accurate from the go-live date forward.
