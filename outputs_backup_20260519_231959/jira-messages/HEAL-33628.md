# Jira Message Draft

Hi Julia,

Great news — I've identified the root cause of the consent signing failure and we've tested the fix successfully.

**Root cause identified:**
Portal users didn't have READ permission on the `Patient_Agreement__c` custom object in Salesforce. When patients attempted to log in and access their consent documents, the portal couldn't retrieve the agreement records — resulting in the red error and blocked signing workflow.

**Fix applied & tested:**
I've updated the "Portal - Authenticated User" profile to grant READ access to:
- Patient_Agreement__c (stores patient consent documents)
- Patient_Agreement_Configuration__c (stores consent type configuration)

This was deployed to our test environment and verified end-to-end. All tests passed — portal users can now query and access agreement records without permission errors.

**What's next:**
The same profile update is ready to deploy to your Production Salesforce environment. Once deployed, patients should be able to:
1. Receive the "Send Patient Agreements" email (already working)
2. Log in to the consent portal with their credentials
3. View and sign their consent documents without the red error

**Important notes:**
- **No Production changes have been made yet** — we're ready to deploy when you approve
- **Metadata change:** Only the Portal - Authenticated User profile is updated (READ-only access)
- **Rollback:** If needed, we can easily revert by removing these permissions
- The workaround your team has been using will continue to work after this fix

**Ready to proceed?**
Please confirm you'd like us to deploy this fix to Production. Once approved, the fix should resolve the issue for all affected patients — and new agreement requests will work immediately.
