## Root Cause

**Not yet confirmed — pending metadata retrieval.** Two primary hypotheses:

1. **(Most likely) Profile or Permission Set change** — A recent admin change removed a permission from Tyler Kaplan's profile or an assigned permission set that previously enabled the native Salesforce "Recall" action in the Approval History related list on Opportunity records. This is consistent with Tyler losing the button without any explicit notification and with OP team members still being able to recall (they likely have a broader permission set such as "Modify All Data" or "Modify All" on Opportunity).

2. **Approval Process `allowRecall` flag disabled** — The active Opportunity Approval Process may have had "Allow submitters to recall approval requests" unchecked during a recent approval process edit. If this were the case, it would affect all non-admin submitters universally. Less likely since OP can still recall (they likely have MAD override), but this setting should still be confirmed.

The "Recall" button in Salesforce Approval History is controlled by two factors:
- The `allowRecall` checkbox on the Approval Process definition
- The submitting user having either the "Modify All Data" permission OR being the original submitter when `allowRecall = true`

Tyler can no longer see it; OP can still action it. This points to either Tyler's permissions being reduced OR the approval process setting being disabled (with OP having MAD as a bypass).
