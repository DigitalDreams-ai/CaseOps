- Summary: Wellvi Submission error
- Reporter: KKrupnik@10xhealthsystem.com

## Root Cause

The Wellvi pharmacy API enforces a hard character limit of **125 characters (including spaces)** on the `RxClaim.Instructions` field in its API payload. The Salesforce org has no corresponding validation rule or Apex-level guard to prevent users from entering or submitting instruction values that exceed this limit. When the order is submitted, the Wellvi API returns `HTTP 422 UnprocessableEntity` with `VALIDATION_ERROR: RxClaim.Instructions- must be 125 characters or fewer`, and Salesforce surfaces this to the user as "X pharmacy order failed to submit."

The reporter (Casey Krupnik) states they already reduced the character count, but the order on `801Ql00000zF7HPIA0` is still failing. This suggests either: (a) spaces are not being counted and the value is still >125 including spaces, or (b) the Salesforce integration layer (Apex callout) appends additional text to the field value before sending to Wellvi, causing the combined string to exceed 125 characters even when the raw field appears short enough.

## Solution Or Escalation

**Immediate (Support/Operator — no deploy required):**
1. Open Order `801Ql00000zF7HPIA0` in Production.
2. Identify the Instructions field (the one mapping to `RxClaim.Instructions` in the Wellvi API payload).
3. Count the current value's character length **including spaces** (e.g., using `=LEN()` in Excel or wordcounter.net).
4. Confirm the value is ≤ 125 characters. If not, reduce it. If it already appears ≤ 125, escalate to Engineering to inspect whether the Apex callout is appending additional text.
5. Save and resubmit the order.

**Durable (Engineering escalation required):**
- Engineering must identify the exact field API name on the Order object that maps to `RxClaim.Instructions`.
- Engineering must confirm whether the Apex callout class concatenates additional text onto this field value before sending.
- Engineering should implement one or both of: (a) a Validation Rule on the Order object blocking saves where Instructions exceeds 125 characters, with a user-friendly message; (b) an Apex-level guard in the callout class that throws a descriptive exception before the API call if the field value exceeds 125 characters.
