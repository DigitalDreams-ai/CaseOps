## Root Cause

There are two compounding issues blocking the Wellvi pharmacy order submission for Opportunity `006Ql00000d7V5RIAU` (patient Jeff Goodwin, CA):

**Issue 1 — STATE_RESTRICTED API error (primary blocker):**
Wellvi's API is returning `UnprocessableEntity: STATE_RESTRICTED` for a CA patient order. Per the reporter (Kaci), CA is only restricted for Tirzepatide — TESA orders to CA should be permitted. The most likely root cause is that Wellvi's account-level configuration for 10x Health incorrectly applies a CA restriction to all products instead of only Tirzepatide. However, it is also possible that the Salesforce integration callout is sending an incorrect product identifier or routing flag that causes Wellvi to apply the wrong restriction rule on their end.

**Issue 2 — Character count rejection (secondary blocker):**
A separate field in the order submission has a character-count constraint. Kaci reduced the field content to 103 characters (including spaces), below the previously understood 125-character limit, yet the order still fails on this basis. This suggests either: (a) Wellvi has tightened or enforces a different limit than 125, (b) the character-count methodology differs (e.g., Wellvi counts differently than Salesforce displays), or (c) the Salesforce callout is not correctly truncating or constructing the field before sending.

**Priority of root cause:**
The STATE_RESTRICTED error appears to be Wellvi-side configuration. The character-count error may also be Wellvi-side or Salesforce-side. Both need vendor confirmation before Engineering is engaged.
