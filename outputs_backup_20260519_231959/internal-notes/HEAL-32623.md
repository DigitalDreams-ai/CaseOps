# Internal Notes

## Jira Issue

- Key: HEAL-32623
- Summary: DEA and DL don't populate in the pharmacy order
- Reporter: SARAH COURCHIA

---

## Root Cause

The autolaunched Salesforce Flow **`Opportunity_to_Order`** (a subflow of `Screenflow - Confirmation to Create Pharmacy Order`) has an incomplete assignment step. The **`Get Opportunity Fields`** element correctly queries `DEA_Number__c`, `State_License_Number__c`, and `Patient_Driver_s_License__c` from the source Opportunity record. However, the downstream **`Assign Order Fields`** assignment element never maps those queried values onto the new Order record before it is saved.

As a result, the Order is created with blank values for these fields, and the "Purchase Order with Scripts" PDF template renders a blank DEA Number on page 1 (and likely blank DL/State License as well).

This is a **regression** — the same issue was fixed in HEAL-30582 but the fix was subsequently overwritten, most likely by a deployment of an older flow version that did not include the corrected assignments.

Root cause confirmed by Sean Bingham (Salesforce Administrator) on 2026-03-24 via direct flow inspection.

---

## Solution Or Escalation

**Engineering escalation required.** The fix requires a Flow Builder change:

1. Open `Opportunity_to_Order` autolaunched flow in Flow Builder.
2. In the `Assign Order Fields` assignment element, add the following three missing mappings:
   - `Order.DEA_Number__c` ← queried Opportunity `DEA_Number__c`
   - `Order.State_License_Number__c` ← queried Opportunity `State_License_Number__c`
   - `Order.Patient_Driver_s_License__c` ← queried Opportunity `Patient_Driver_s_License__c`
3. Save and activate the new flow version in Sandbox.
4. Validate by generating a pharmacy order from an Opportunity with all three fields populated; confirm fields appear on the Order and in the PDF.
5. Deploy to Production via Gearset.

Additionally, Engineering should investigate **why the HEAL-30582 fix was overwritten** to prevent a third recurrence. A deployment checklist or version-locking strategy for this flow is advisable.
