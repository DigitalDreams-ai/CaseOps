## Root Cause

The `Supplement Inquiry` picklist field (`Supplement_Inquiry__c`) was never placed on the Case page layout used by the Customer Support team, and Field-Level Security (FLS) was not configured for the relevant permission sets. The field itself already existed in Production (pre-created) but was inaccessible to Customer Support agents because it was absent from the **Case-Customer Experience** layout and lacked FLS entries. This is a pure declarative configuration gap — no code, Flows, or Validation Rules are involved.

Secondary note: the original request listed "Precision Weight Management Supplements 10X 5-MTHF" as what appeared to be a single run-together entry. Based on the form's bullet-list formatting, these are two separate picklist values: "Precision Weight Management Supplements" and "10X 5-MTHF". All 36 values are confirmed active in the field definition.

## Solution Or Escalation

**Support-resolvable — no Engineering escalation required.**

Sandbox fix applied:
1. Added `Supplement_Inquiry__c` to the **Specialties section** of the **Case-Customer Experience layout**, positioned next to `Call_Details__c`, with Edit behavior (visible and editable).
2. Added an FLS entry to the **System_Administrator permission set** granting `readable=true` and `editable=true` for `Case.Supplement_Inquiry__c`, matching the established pattern of similar picklist fields (`Inquiry_Category__c`, `Dispo_Level_1__c`, `Case_Resolution_Type__c`).
3. All 36 picklist values confirmed active in the field definition (already present in Production and Sandbox).
4. Madison Veals and Christine Greig should be added as watchers on Jira ticket HEAL-33659 — this is an immediate administrative action independent of the Salesforce deployment.

Sandbox deployment validated successfully (Deploy ID: `0AfEa00000ZxSFRKA3`, 2/2 components, 6.86 seconds). No breaking changes observed.
