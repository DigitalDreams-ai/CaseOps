## Root Cause

Sarah Phonthaphanh and three named colleagues (Kaci Krupnik, Sarah Courchia, Kristhal Valdez) on the order processing management team lack sufficient permissions to view all Order records in Salesforce. The root cause is most likely one or a combination of:

1. **Missing Permission Set / Profile gap:** Their current Profile does not grant broad Read access to the Order object beyond records they own, and no Permission Set supplements this.
2. **Missing or inaccessible "All Orders" list view:** No list view exists (or is shared with them) that surfaces all Order records — they default to the `__Recent` filter, which only shows recently accessed records.
3. **OWD restriction:** If the Order object OWD is set to Private or Public Read Only scoped by ownership, users outside the owner role hierarchy cannot see records they don't own.

This is purely a configuration/access issue — no Apex, flows, or automation are involved.

## Solution Or Escalation

**Support-resolvable.** Proposed steps:

1. **Audit** the 4 users' current Profiles, Permission Set assignments, Role Hierarchy placement, and the Order object OWD in Production.
2. **Verify or create an "All Orders" list view** on the Order object with appropriate sharing (e.g., shared with a Public Group: Order Processing Managers, or All Internal Users if appropriate).
3. **Verify or create a Permission Set** (e.g., `Order_Processor_Lead_Access`) granting at minimum Read on the Order object — validate in Sandbox first if creation is needed.
4. **Assign the Permission Set** to Sarah Phonthaphanh, Kaci Krupnik, Sarah Courchia, and Kristhal Valdez.
5. **If OWD is Private:** Create a Sharing Rule granting Order visibility to the manager/lead tier via Role or Public Group, so they can see records owned by their direct reports.
6. Confirm with Sarah whether scope is org-wide orders or team-scoped orders before finalizing.
