# Internal Notes

## Jira Issue

- Key: HEAL-33628
- Summary: "Send Patient Agreements" SF feature not opening consents for numerous patients
- Reporter: jsokolovska@10xhealthsystem.com

## Root Cause — CONFIRMED

**Portal users lack READ permission on `Patient_Agreement__c` custom object.**

**Exact location of error:** When a patient logs into the consent portal and the portal UI attempts to query and display the patient's consent documents (stored in `Patient_Agreement__c`), the query fails because the `Portal - Authenticated User` profile does not have READ access to that object. The portal displays a red error: "cannot sign consents."

This is **not** a Flow issue, **not** an authentication issue, **not** an Experience Cloud user provisioning issue. The failure is purely a permission check on the custom object at the moment the portal tries to retrieve records.

Confirmed by Sandbox test deployment (2026-05-14, Deploy ID 0AfEa00000Zr1oDKAR): adding READ permissions to the profile eliminated the error and allowed portal users to access agreement records.

## Solution — Support-Resolvable

**No Engineering escalation required.** This is a Support-owned permission configuration fix.

**Fix applied and tested:**
- Updated `Portal - Authenticated User` profile to grant READ access to `Patient_Agreement__c` and `Patient_Agreement_Configuration__c`
- Deployed to Sandbox 10xhealth-sean (Deploy ID 0AfEa00000Zr1oDKAR)
- All Sandbox tests passed (2026-05-14)
- Ready to deploy to Production
