# Internal Notes

**Reference:** See `outputs/investigations/HEAL-33098.md` for full problem analysis, Salesforce configuration, and similar items analysis.

## Root Cause

The SMS Magic trial license has expired in the lower Salesforce environments (QA, UAT, DEV). This is a third-party managed package (SMS Magic / Converse) licensing issue — the sandbox/lower-environment licenses are not active, causing the "trial expired" banner and preventing messages from being sent or delivered. Additionally, the reporter's own Salesforce user account does not have an SMS Magic license assigned, meaning even if the org-level license were active, their user could not use the feature. This is a vendor license and configuration issue, not a Salesforce metadata or code defect.

## Solution Or Escalation

**Escalation required — to SMS Magic vendor and/or internal Salesforce Admin with managed package admin rights.**

This is not solvable via a Sandbox metadata deploy or code change. The resolution requires:

1. **Contact SMS Magic support** to extend or activate sandbox/lower-environment licenses for the QA, UAT, and DEV orgs. SMS Magic typically offers sandbox licenses separately from production licenses — this needs to be requested and provisioned through the vendor portal or their support team.
2. **Assign SMS Magic user licenses** within each lower environment to the appropriate engineering and QA team members (done via Salesforce Setup → Installed Packages → SMS Magic → Manage Licenses).
3. **Validate UAT integration end-to-end** after licenses are activated, confirming messages are sent and received (minimum requirement per the reporter).
4. **Verify phone number / messaging channel configuration** in lower envs — even with a valid license, sandbox environments may require a separate sandbox-configured sending number or API key from SMS Magic.

No Salesforce custom metadata, Apex code, or declarative configuration changes are needed from the CaseOps pipeline.
