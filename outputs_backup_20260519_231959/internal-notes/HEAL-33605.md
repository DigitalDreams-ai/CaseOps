## Root Cause

Amy Stolze is missing one or more Permission Set assignments that grant access to email and SMS templates in Salesforce. The reporter confirmed she has zero access to any templates and requested that her access mirror that of an existing user, Veronica B. The most likely root cause is that when Amy's user account was created or configured, the relevant Permission Set(s) governing template access were not assigned to her, while Veronica B has them. A secondary contributing factor may be email template folder sharing (Lightning email templates are folder-scoped). The SMS template tool/package is not yet identified — if it is a managed package, a package license seat may also be required.

This is not a code, flow, or automation issue. It is a pure access/permissions configuration gap.

## Solution Or Escalation

**Support-resolvable.** Steps:
1. Run SOQL in Production to identify Amy Stolze's User ID and current Permission Set assignments.
2. Run SOQL in Production to identify Veronica B's User ID and Permission Set assignments.
3. Diff the two lists — identify Permission Sets assigned to Veronica B that Amy is missing.
4. Review the scope of those Permission Sets to avoid overpermissioning (particularly if they cover more than just templates).
5. Assign the appropriate Permission Set(s) to Amy Stolze directly in Production (Setup → Users → Amy Stolze → Permission Set Assignments).
6. Check email template folder sharing in Production — grant Amy access to the same folders Veronica B can access.
7. If SMS templates are managed-package-driven, confirm a license seat is available and assign it alongside the Permission Set.
8. Validate by using "Login As" (if enabled) or by asking Amy to test access and confirm.
9. Resolve HEAL-33605 once Amy confirms access.
