## Jira Issue

- Key: HEAL-33647
- Summary: Shipping Address Mismatch - Shopify vs Salesforce
- Reporter: Nathaniel Cooper

## Root Cause

The direct root cause is that the `Shipping Same As Billing Address` field on Heatherly Grace Shepherd's Person Account (`001Ql00000xapgPIAQ`) is set to **TRUE**. This caused Salesforce to treat the Shipping Address as identical to the Billing Address, resulting in the customer's supplement order being delivered to the wrong address.

The Salesforce-internal automation has been cleared:
- `Autolaunch_Account_Sync_To_External_Systems` flow: reviewed in Production — correctly maps billing fields to `billingAddress` and shipping fields to `shippingAddress`. Not the cause.
- `Patient_Patient` Apex action: reviewed in Production — does not update Account records and does not map billing fields into shipping fields. Not the cause.

**Unanswered question (RCA gap):** How did `Shipping Same As Billing Address` get set to TRUE? The Shopify-to-Salesforce connector has not been reviewed (Sean Bingham does not have access). This is the most likely remaining candidate for the root cause, particularly if the connector sets this flag during customer/order record creation or update.

## Solution Or Escalation

**Immediate fix (Support-resolvable — Production data correction):**
1. On Person Account `001Ql00000xapgPIAQ` in Production, set `Shipping Same As Billing Address` = FALSE.
2. Update the Shipping Address fields to match the correct Shipping Address from Shopify order `6671104344278`.

**RCA (requires connector owner / Engineering):**
3. Identify who owns the Shopify-to-Salesforce connector and request a review of its field mappings, specifically whether it writes `Shipping Same As Billing Address = TRUE` under any conditions.
4. If systemic, run a SOQL query to identify all other affected Person Accounts and coordinate a bulk data correction.
5. If a connector mapping bug is confirmed, Engineering must correct the mapping to prevent recurrence.
