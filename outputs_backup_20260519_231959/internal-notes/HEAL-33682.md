## Root Cause

**Not yet fully confirmed — requires Engineering investigation.** Based on available evidence:

The Novogenia integration layer (Apex, Flow, or middleware) included product code `#VRSF - Report Print & Shipment Fee` (PGT Booklet printed report) in the outbound API payload submitted to Novogenia for three Lab Orders. Novogenia received and fulfilled these booklet orders. However, no corresponding Salesforce record (Lab Order line item, child Lab Order, or Opportunity product) was created in Salesforce to reflect the PGT Booklet component. This is a silent data gap — Novogenia has the order, Salesforce does not.

The most probable cause is that the code or automation path that constructs the Novogenia API payload for `#VRSF` does not include a corresponding Salesforce record-creation step, or that step failed silently. At least two of the three affected orders are from 2024, and Salesforce debug logs are limited to 30 entries, making the original API call payload unrecoverable through standard Support tooling.

3. A potential retroactive data fix (back-filling Salesforce records for the three affected Kit IDs) must be guided by Engineering findings to avoid triggering automations or creating conflicting records.
