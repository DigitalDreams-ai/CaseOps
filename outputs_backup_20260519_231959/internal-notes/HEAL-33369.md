## Root Cause

**Two compounding issues:**

1. **Structural report type limitation:** The reports use the Opportunity Field History report type, which does not natively expose Account-level fields (Billing State, Main Practitioner). This is a standard Salesforce platform constraint — Account fields cannot be directly added to an Opportunity Field History report.

2. **Blank Opportunity-level fields:** There are custom Opportunity fields for "Main Practitioner" and "State" that were presumably intended to mirror Account data (Billing State and Main Practitioner), but these fields are currently blank across relevant records. This means even if the report type could surface these Opportunity fields, they would show no data. Sean's comment on 2026-05-08 confirms he found "existing processes for this that are problematic" — indicating there is already automation (Flow, Process Builder, or Apex) intended to populate these Opportunity fields from the Account that is not working correctly.

The likely failure point: auto-created Lab Consultation Opportunities may be created before or without the Account relationship being fully established, causing any create-trigger automation to fire when Account data is not yet available, resulting in blank fields. Alternatively, the automation may have a logic error, be deactivated, or have a filtering condition that excludes these records.

## Solution Or Escalation

**Recommended approach (Support-resolvable with possible Engineering review):**

1. **Audit existing automation** — Identify all Flows, Process Builder rules, and Apex triggers that reference the Opportunity-level Main Practitioner and State fields. Understand why they are "problematic" (Sean's word). Deactivate or correct any conflicting automation before adding new logic.

2. **Create or fix a Record-Triggered Flow** on the Opportunity object that:
   - Triggers on Create AND Update (when `AccountId` is set or changes)
   - Copies `Account.BillingState` → Opportunity State field (custom)
   - Copies `Account.Main_Practitioner__c` (or equivalent) → Opportunity Main Practitioner field (custom)
   - Handles null Account gracefully

3. **Verify report field availability** — Once Opportunity fields are populated, confirm they surface as available columns in the Opportunity Field History report type. If not, a Custom Report Type (Opportunity Field History → Opportunity relationship) may be needed.

4. **Update all three reports** to add the Billing State and Main Practitioner columns:
   - Lab Review: `00OQl000006nwH7MAI` (highest priority)
   - Genetic Test Clarity Calls: `00OQl000006nyxFMAQ`
   - Peptide Express Consultations: `00OQl000006nz0TMAQ`

5. **Assess backfill need** — Historical Opportunity records will have blank fields. A one-time data update (Data Loader or batch Flow) may be needed to backfill existing records so historical TTS data is usable by state/provider.

**If "existing problematic processes" are Apex-based:** Escalate that component to Engineering for review/fix. The report and Flow components can still be handled by Support/Admin.
