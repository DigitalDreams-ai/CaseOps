## Root Cause

Salesforce's standard Case object does not record a timestamp when a Case is escalated. The standard `IsEscalated` field is a boolean checkbox only — it captures whether a case has been escalated but not when. No custom escalation timestamp field or automation appears to exist in the org (to be confirmed by Engineering via field inventory). As a result, there is no data point available to anchor elapsed-time calculations between Case open → escalation or escalation → close in any Salesforce report.

---

## Solution Or Escalation

**Engineering escalation required.** The solution involves three layers:

1. **Custom Field:** Create `Case.Escalation_DateTime__c` (Date/Time) on the Case object to store the escalation moment.
2. **Record-Triggered Flow:** Automate stamping of `Escalation_DateTime__c = NOW()` when the escalation condition fires (trigger mechanism must be confirmed with Noah Scott — e.g., `IsEscalated` → true, Status change to an escalation value, or queue reassignment to Tier 2).
3. **Reporting:** Build a Cases report using the new field and optionally create formula fields for elapsed-time calculations (`Time_To_Escalate_Hours__c`, `Time_Escalated_To_Close_Hours__c`).

This cannot be resolved by Support alone — requires custom metadata creation and Flow automation.
