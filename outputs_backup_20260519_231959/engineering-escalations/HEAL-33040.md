## Engineering Message

**Issue:** HEAL-33040 — Patient Account not syncing to Shopify; both automatic and manual sync mechanisms broken
**Problem:** A patient Account created in Salesforce via the "new patient link" did not sync to Shopify automatically. The Account (`001Ql00000x8VCkIAM`) has no Shopify Customer ID populated. A manual override field (`Trig Shopify Sync`) was set to `True` by Danielle Cress (Technical Support Manager) as a fallback, and it also did not trigger a sync. Both the automatic creation-time sync and the manual trigger are non-functional.
**Potential Fix:** Identify the Flow and/or Apex trigger responsible for syncing Salesforce Accounts to Shopify on creation and on `Trig Shopify Sync = True`. Investigate why it is failing (e.g., unhandled exception, API authentication failure, inactive Flow, missing field on accounts created via the new patient link). Fix the root cause, redeploy to Production, and manually resync the affected account. Audit for other recently created accounts with the same missing sync.
