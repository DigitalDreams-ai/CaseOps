## Engineering Message

**Issue:** HEAL-33647 — Shipping Address Mismatch - Shopify vs Salesforce

**Problem:** Customer's shipping address was overwritten with billing address in Salesforce during Shopify-to-Salesforce sync, causing a package to be delivered to the wrong address. In Shopify, shipping and billing addresses are correctly distinguished. In Salesforce Account records, the Shipping Address field contains the billing address instead of the correct shipping address.

**Potential Fix:** Debug and fix the custom Apex `Patient_Patient` action or external Shopify connector configuration that receives address data from Salesforce flows. The Salesforce-side flows correctly map shipping and billing addresses separately. The bug is in the external integration layer, not Salesforce automation. Correct the field mappings so Shopify shipping address maps to Salesforce ShippingStreet/ShippingCity/etc., and Shopify billing address maps to BillingStreet/BillingCity/etc., without cross-contamination.

---

## Steps to Reproduce

**Affected record IDs:** 
- Salesforce Account: `001Ql00000xapgPIAQ`
- Shopify Order: `6671104344278`

**Steps:**
1. In Shopify admin, navigate to the order (6671104344278) and confirm shipping address is different from billing address.
2. In Salesforce, open Account record 001Ql00000xapgPIAQ.
3. View the Shipping Address field(s) (ShippingStreet, ShippingCity, ShippingState, ShippingPostalCode).
4. **Observe:** Shipping Address contains the billing address instead of the correct shipping address from Shopify.

## Root Cause

Shopify-Salesforce integration is incorrectly mapping address data. The custom Apex action `Patient_Patient` (or external connector) receives address data from the active Account trigger flow `Record_Trigger_Account_Account_Sync_To_External_Systems`. Salesforce flows are verified correct and map shipping/billing addresses separately. The external integration misconfigures field mappings, causing Shopify shipping address to be written to Salesforce billing address field (or vice versa), or address data is corrupted in transit.

## Affected Component

- Metadata/code component: Custom Apex `Patient_Patient` action or external Shopify connector (Zapier, Make, native Shopify App, or MuleSoft integration)
- Element/rule/process, if applicable: Account address field mapping logic in Apex or connector configuration; specifically, logic that receives Shopify address data and writes to Salesforce Account ShippingStreet/BillingStreet fields

## Potential Fix

1. Audit the custom Apex `Patient_Patient` action to confirm it correctly receives Shopify address data and maps fields without swap or corruption.
2. If external connector (Zapier, Make, etc.), audit field mappings in connector configuration to ensure Shopify shipping → Salesforce Shipping and Shopify billing → Salesforce Billing.
3. Check connector logs or Apex logs for Account 001Ql00000xapgPIAQ to pinpoint when address was overwritten.
4. Once root cause is confirmed, fix the mapping logic and re-test with Account 001Ql00000xapgPIAQ.
5. Conduct RCA query: identify all Accounts where ShippingAddress == BillingAddress when they should differ (scope of issue). Flag any other affected customers.
