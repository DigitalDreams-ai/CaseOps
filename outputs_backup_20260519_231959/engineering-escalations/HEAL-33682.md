# Engineering Handoff

## Engineering Message

**Issue:** HEAL-33682 — Orphan PGT Booklets- No order on SF

**Problem:** Novogenia received and fulfilled PGT Booklet (printed report) orders for three Lab Orders, but Salesforce contains no corresponding Lab Order line item, child Lab Order, or Opportunity product record reflecting the PGT Booklet (`#VRSF - Report Print & Shipment Fee`) for these orders. Novogenia confirmed via their Portal that `#VRSF` was included in the original API request for all three Lab Orders; Salesforce has no Salesforce-side record of the Booklet being ordered.

**Potential Fix:** Identify the Salesforce Novogenia integration code (Apex, Flow, or middleware) that constructs the outbound API payload and appends `#VRSF` to the request. The payload is reaching Novogenia correctly, but the Salesforce-side record creation step for `#VRSF` is missing, failing silently, or not implemented. Fix: (1) Locate the code path that includes `#VRSF`; (2) add or repair the Salesforce record creation step to ensure a Lab Order line item or child record is created when `#VRSF` is appended; (3) backfill missing records for the three affected Lab Orders if feasible; (4) audit for additional orphaned booklet orders beyond the three identified.

## Root Cause

**High-confidence hypothesis from SOQL data audit (2026-05-19):**

The Novogenia integration code includes logic to append `#VRSF` (PGT Booklet product) to outbound API payloads when certain conditions are met, **but the corresponding Salesforce record-creation step either:**

1. **Does not exist** — the integration calls out to Novogenia with `#VRSF` in the payload but has no code path to create an OpportunityLineItem or Lab Order product record in Salesforce.
2. **Exists but is broken or disabled** — a Flow action or Apex method to create the record fails silently, is wrapped in a try-catch that suppresses errors, or is conditionally disabled.
3. **Fails only under specific conditions** — the record-creation logic may have a bug or missing field mapping that causes it to fail for the `Printed Report Books` product specifically, while the Novogenia callout succeeds.

**Supporting evidence from SOQL audit:**
- **Three Lab Orders confirmed:** LAB-000228814 (03/02/2026), LAB-000232068 (04/07/2026), LAB-000058193 (05/27/2024). All exist in Production; all linked to correct Opportunities.
- **Three Opportunities confirmed:** Each contains ONE OpportunityLineItem for the main PGT product (Product2 ID `01tRh000003qAcfIAE`, list price $1299). **None have the "Printed Report Books" product (Product2 ID `01tRh000006VKS1IAO`).**
- **Product mismatch:** Salesforce has a "Printed Report Books" product (`01tRh000006VKS1IAO`, the semantic equivalent of Novogenia's `#VRSF`), but it has no ProductCode field, no Pricebook entries, and no OpportunityLineItem linking it to the affected Opportunities.
- **Product code missing:** Query `SELECT * FROM Product2 WHERE ProductCode LIKE '%VRSF%'` returned 0 records. The `#VRSF` code does not exist in Salesforce Product2 at all.
- **No junction object:** `Lab_Order_Product__c` does not exist. Lab Orders support only a single `Product__c` field, not a collection of products. No child object exists for multi-product orders.
- **Integration code not found locally:** Novogenia integration is Production-only or in a managed package; cannot be reviewed in checked-in repo.
- **Nebula Logger available:** Actively used by Flows (confirmed in `Autolaunch_Create_Order_for_Fulfilled_By_Items.flow`); logs for the affected Lab Orders exist but require browser UI access to retrieve.
- **Novogenia Portal confirms `#VRSF` fulfilled:** Courtney Marlin verified that `#VRSF - Report Print & Shipment Fee` is listed as a fulfilled product on all three orders in Novogenia's external system.

**Most likely scenario:** The integration was modified to send `#VRSF` to Novogenia (perhaps a recent change), but the corresponding Salesforce record-creation code either was not written or was added in one branch but not merged. Novogenia accepts the `#VRSF` product unconditionally, so the API succeeds; Salesforce never records the order because the code path doesn't exist or is broken.

## Potential Fix

1. **Log access (highest priority):** Access Nebula Logger, middleware logs, or Event Log Files in Production to retrieve the original Novogenia API call payload for the three Lab Order IDs. Confirm that `#VRSF` was included in the request and identify the code path that executed the callout.
2. **Code review:** Once the code path is identified, review the Apex class(es), Flow(s), or middleware responsible for:
   - Constructing the Novogenia API payload and appending `#VRSF`
   - Creating or updating Salesforce Lab Order and related records (Opportunity, OpportunityLineItem) to reflect the PGT Booklet
   - Determine whether a record-creation step is missing, disabled, or failing silently
3. **Fix implementation:** Add or repair the missing record-creation step to ensure that whenever `#VRSF` is included in the Novogenia payload, a corresponding Lab Order line item or related record is created in Salesforce with the correct product reference, pricing, and parent-child relationships.
4. **Data backfill:** Determine whether retroactive Salesforce record creation is feasible for the three known orphaned orders. Validate that backfilling will not trigger unintended automations, notifications, or billing processes.
5. **Audit:** Run a broader query or data audit to identify any additional orphaned PGT Booklet orders beyond the three flagged here. This issue may affect multiple orders, especially if it is a legacy integration behavior from 2024.
