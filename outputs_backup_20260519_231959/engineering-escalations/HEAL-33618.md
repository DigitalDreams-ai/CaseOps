# Engineering Handoff: HEAL-33618

## Engineering Message

**Issue:** HEAL-33618 — CX Case function reporting  
**Problem:** Salesforce's standard Case object does not record a timestamp when a Case is escalated. The standard `IsEscalated` field is a boolean only — it captures whether a case has been escalated but not when. Without an escalation timestamp stored on the Case record, there is no data point to calculate elapsed time between Case open → escalation or escalation → close in any Salesforce report.  
**Potential Fix:** Create a custom Date/Time field (`Escalation_DateTime__c`) on the Case object. Build a Record-Triggered Flow to stamp this field at the moment of escalation. Optionally create formula fields for elapsed-time calculations and a report surfacing the new fields.

---

## Steps to Reproduce

**Affected record IDs:** All Cases in the `Customer_Success_Team_Case` list view escalated from Tier 1 to Tier 2 (no specific record IDs provided by requester).

**Steps:**
1. Log in to `10xhealth.lightning.force.com` as a Customer Support Tier 1 or Tier 2 agent.
2. Navigate to Cases → `Customer_Success_Team_Case` list view.
3. Open any Case that has been escalated from Tier 1 to Tier 2, or simulate escalation by triggering the current escalation mechanism (e.g., checking `IsEscalated`, changing Status, or reassigning to Tier 2 queue).
4. Observe: No Date/Time field on the Case record captures when the escalation occurred. Attempting to build a report with elapsed time from escalation is impossible — no timestamp field exists.

**Expected behavior:** A Date/Time field (`Escalation_DateTime__c`) is stamped automatically when the Case is escalated, enabling elapsed-time reporting in both directions (open → escalated, escalated → closed).

## Root Cause

This is NOT a permissions issue, NOT a broken existing field, and NOT a report configuration error. The root cause is a **product gap**: Salesforce's standard Case object does not record a timestamp for escalation events. The `IsEscalated` field is a boolean only. No custom escalation timestamp field or automation exists in the org (to be confirmed). Engineering must design and implement the data capture layer before reporting is possible.

---

## Affected Component

- **Metadata/code component:** `Case` object (standard)
- **Elements to be created:**
  - `Case.Escalation_DateTime__c` — custom Date/Time field
  - `Case.Time_To_Escalate_Hours__c` — optional formula field (elapsed hours from CreatedDate to Escalation_DateTime__c)
  - `Case.Time_Escalated_To_Close_Hours__c` — optional formula field (elapsed hours from Escalation_DateTime__c to ClosedDate)
  - Record-Triggered Flow on Case object — fires on escalation trigger condition, stamps `Escalation_DateTime__c = NOW()` (only when null, to preserve first escalation)
  - Cases report — new report surfacing CreatedDate, Escalation_DateTime__c, ClosedDate, and elapsed-time calculations

---

## Potential Fix

1. **Custom Field:** Create `Case.Escalation_DateTime__c` (Date/Time) to store the moment of escalation.
2. **Record-Triggered Flow:** Create a Record-Triggered Flow on the Case object that fires when the escalation condition is met. Set `Escalation_DateTime__c = {!$Flow.CurrentDateTime}` if the field is currently blank (null check — preserve first escalation only).
   - **Blocker:** The escalation trigger mechanism must be confirmed with Noah Scott (e.g., `IsEscalated` changed to `true`, Status changed to an escalation value, or assignment to Tier 2 queue). This is the single most important unknown.
3. **Formula Fields (optional but recommended):**
   - `Time_To_Escalate_Hours__c`: `(Escalation_DateTime__c - CreatedDate) * 24` — returns hours.
   - `Time_Escalated_To_Close_Hours__c`: `(ClosedDate - Escalation_DateTime__c) * 24` — returns hours (meaningful when case is closed).
4. **Report:** Build a Cases report surfacing `CreatedDate`, `Escalation_DateTime__c`, `ClosedDate`, and the two elapsed-time formula fields. Scope to the `Customer_Success_Team_Case` filter.
5. **Backfill (if needed):** Confirm with stakeholder whether historical Cases need a retroactive escalation date (likely not possible without audit log review).
