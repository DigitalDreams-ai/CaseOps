# Engineering Handoff

## Engineering Message

**Issue:** HEAL-33116 — Adding "Scheduled by" into Consult Opp (Calendly Automation — Deferred Scope)
**Problem:** A new picklist field `Scheduled_By__c` ("Scheduled by") has been added to the Opportunity object in Salesforce by the SF Admin team. The field is intended to track who scheduled a consultation appointment. When a booking is made via Calendly and syncs to Salesforce as a scheduled Opportunity, the field should automatically default to "Self-Scheduled". This automation does not yet exist — the field is currently populated manually only.
**Potential Fix:** Implement an automation (Record-Triggered Flow, Apex Trigger, or update to the existing Calendly→Salesforce integration layer) that sets `Scheduled_By__c` = "Self-Scheduled" when a Calendly booking event creates or updates a Consultation Opportunity in Salesforce. The field should remain editable post-creation so schedulers can override it if manual intervention occurred.

---

## Steps to Reproduce

**Affected record IDs:** Example Opportunity: `006Ql00000bJEWbIAO` (referenced by reporter as a consult opp example)

**Steps:**
1. Log in to Salesforce as an Operations or Admin user.
2. Navigate to a Consultation Opportunity that was created via a Calendly booking sync (e.g., `006Ql00000bJEWbIAO` or any recent Calendly-sourced Opp).
3. Open the Opportunity record and locate the "Scheduled by" field (`Scheduled_By__c`).
4. Observe: The "Scheduled by" field is blank — it is not automatically populated with "Self-Scheduled" even though the booking originated from Calendly.

**Expected behavior:** When a Calendly booking syncs to Salesforce and creates/updates a Consultation Opportunity with a scheduled status, `Scheduled_By__c` should be automatically set to "Self-Scheduled". Schedulers should be able to manually change this value afterward if they intervened.

## Root Cause

The Calendly-to-Salesforce integration (sync mechanism TBD — likely native Calendly SF integration, Zapier, or custom middleware) does not currently set any value on the new `Scheduled_By__c` field when it creates or updates Opportunity records. The field was added after the integration was established, so no automation exists to populate it. The SF Admin team confirmed this automation was intentionally deferred to Engineering/Dev scope during the initial field creation work.

## Affected Component

- **Metadata/code component:** `Opportunity.Scheduled_By__c` (Picklist field — already exists in Salesforce)
- **Integration layer:** Calendly → Salesforce sync (integration mechanism needs Engineering investigation — could be native Calendly SF package, Zapier, custom Apex, or Flow)
- **Element/rule/process, if applicable:** New Record-Triggered Flow or update to existing Calendly integration handler to set `Scheduled_By__c` = "Self-Scheduled" on Calendly-sourced Opportunity records

## Potential Fix

1. **Identify the Calendly→Salesforce integration mechanism** (native package, middleware, Apex, or Flow) — determine where Opportunity records are created/updated when a Calendly booking occurs.
2. **Add automation to set `Scheduled_By__c` = "Self-Scheduled"** at the point of Calendly-triggered Opportunity creation/update. Options:
   - If using a Record-Triggered Flow: Add a condition checking for a Calendly source indicator and set the field value on creation.
   - If using Apex: Update the relevant trigger/handler class to assign the picklist value.
   - If using middleware (Zapier/custom): Add a field mapping for `Scheduled_By__c` = "Self-Scheduled" in the Calendly → Salesforce action.
3. **Ensure the field remains editable** after auto-population so schedulers can override to another picklist value (Kershon Stephens, Margo Michel, Monica Lewis, Jemimah Aquino, Stephannie Opeda, Mariella Medina, Management, Other) when manual intervention occurred.
4. **Do not overwrite existing non-blank values** — the automation should only set the default if the field is currently null/blank (to preserve manual entries).
