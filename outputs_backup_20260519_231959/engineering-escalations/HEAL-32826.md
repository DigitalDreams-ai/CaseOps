# Engineering Handoff: HEAL-32826

## Engineering Message

**Issue:** HEAL-32826 — PAM Informed Consent Cases for Shopify Orders missing backfill after sync failure

**Problem:** Between 3/18–3/27/2026, several hundred Salesforce Account records for Shopify MGT/GHT orders failed to sync "Informed Consent 2026" from Shopify. A fix deployed 3/27 resolved the issue for new orders, but ~300+ affected accounts remain without consent data. Additionally, the consent removal workflow (SF → Chrono → removed from SF) is not occurring, unclear if design intent or missing feature.

**Potential Fix:** (1) Audit Account/Opportunity creation flow/Apex to identify why consent sync failed 3/18–3/27; (2) verify 3/27 fix is permanent; (3) clarify consent removal design and implementation status; (4) validate "Assign patient agreements" bulk action for safe backfill and provide remediation scope.

---

## Steps to Reproduce

**Affected record IDs:** Approximately 300+ Accounts and Cases; example Case: 500Ql00000tN0qrIAC

**Steps:**
1. Log in to Salesforce Production as System Admin
2. Query Accounts where Opportunities have CreatedDate between 2026-03-18 and 2026-03-27 AND Account.Informed_Consent_2026__c IS NULL
3. Observe: "Informed Consent 2026" field is empty for these accounts
4. Create a test Shopify MGT order in current environment (post-3/27 fix)
5. Observe: Informed Consent field auto-populates on Account within minutes

**Expected behavior:** Consent assigned in Shopify → synced to Salesforce Account/Opportunity on record creation within minutes → forwarded to Chrono → removed from Salesforce (or retained per design clarification)
