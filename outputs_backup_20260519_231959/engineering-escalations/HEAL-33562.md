## Root Cause

**Investigation Dead-end:** Support identified "ApprovalApprove" system permission as missing from Clinical profile and nursing permission sets. However, "ApprovalApprove" is not a valid Salesforce system permission—deployment to Sandbox rejected as unknown permission.

**Actual Root Cause (Unknown):** Permission or configuration difference between Clinical and Order Processing users. Possibilities include:
- Custom metadata configuration controlling recall visibility by role/department
- Approval process custom logic or flow conditions restricting recall by user role
- Undocumented permission or system setting not exposed in standard UI
- Role-based approval configuration in Opportunity_Approval_Process
