## Root Cause

SMS notification routing logic (Flow or Process Builder) uses incorrect field mapping or hardcoded values to determine notification email recipient. When a patient replies to an SMS sent by Jemimah, the notification is routed to Steph instead of Jemimah because the flow does not correctly reference the actual SMS sender field.

Possible causes:
1. Flow/Process uses `CreatedBy` or `Owner` field instead of actual sender field
2. Vonage webhook mapping hardcodes recipient or uses incorrect user lookup
3. Email template contains hardcoded recipient formula that doesn't account for sender identity
4. Custom sender field exists but is not referenced in flow/process logic
