**Issue:** HEAL-33605 — Amy Stolze needs access to email and SMS templates in Salesforce

**Problem:** Amy Stolze (Wellness, Concierge user) has zero access to email and SMS templates in Salesforce, while peer Veronica Berenguer has full access. Permission set investigation revealed both users have identical assignments, including SMS_App_Permission_Set which grants full CRUD on the custom tdc_tsw__Message_Template__c object. ## Root Cause Analysis

Permission set assignment hypothesis **REJECTED by metadata verification**:

- Amy Stolze: Concierge profile + 8 permission sets (Agent, Agent_10XHS, Campaign_Launcher, SMS_App_Permission_Set, SMS_Interact_Conversation_User, SMS_Magic_Additional_Permissions, Wellness_Concierge, and profile permission set)
- Veronica Berenguer: Concierge profile + **identical 8 permission sets**
- SMS_App_Permission_Set grants full CRUD on tdc_tsw__Message_Template__c (confirmed in metadata)
- Concierge profile contains no object-level permissions; all access via permission sets

**Conclusion:** Both users have identical profiles and permission sets including SMS template access. Access gap is not due to missing permission assignments. Root cause must be elsewhere:

1. **User-level configuration:** Amy's user record may have blocking flags (status disabled, IP restriction, login method incompatibility, MFA state, org limit flags)
2. **Org-wide defaults or sharing rules:** Template object may have org-wide defaults or sharing rules limiting visibility (e.g., role-based, field-based, or custom sharing rules)
3. **Custom permissions or org settings:** Org hierarchy, custom settings, or feature toggles may gate template access
4. **Record/folder-level security:** Template records may have folder permissions or custom record-level sharing rules
5. **Custom code or flow logic:** SMS app or custom Apex may have programmatic access control that blocks Amy
