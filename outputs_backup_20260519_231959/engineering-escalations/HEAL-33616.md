# Engineering Handoff: HEAL-33616

## Engineering Message

**Issue:** HEAL-33616 — Salesforce Lab Order error: Update Order flow fails with "Please insert a category or a product!"

**Problem:** When PAM staff click the "Update Order" button on Novogenia Lab Orders to resubmit after updating patient demographics (height/weight), the flow returns error: "errorMessage: Please insert a category or a product!" (HTTP 400). This affects multiple Lab Orders (at least 2 confirmed).

**Potential Fix:** The Update Order flow is missing required category or product field mappings when building the API payload to Novogenia. Review the flow''s field-mapping step; ensure Product__c and Category fields (or Novogenia API contract equivalents) are populated before API call. Add error handling for missing required fields.

## Root Cause

The "Update Order" flow (targeting Novogenia provider) is building an incomplete API payload for the Novogenia integration. Either:
1. Product__c field is null/not being fetched from the Lab Order record
2. Category field mapping is missing (check what Novogenia API expects)
3. Flow condition is filtering out these fields incorrectly
4. Novogenia API contract changed and requires a new field

The fact that two separate Lab Orders exhibit the same error indicates a systemic flow logic issue, not isolated data corruption.

## Affected Component

- **Flow:** "Update Order" flow (likely on Lab_Order__c object)
- **Provider:** Novogenia (this button is Novogenia/3PL only per Lab Order SOPs)
- **Integration:** Novogenia API call for order resubmission
- **Fields involved:** Product__c, Category (or equivalent API contract fields), Patient Height/Weight
