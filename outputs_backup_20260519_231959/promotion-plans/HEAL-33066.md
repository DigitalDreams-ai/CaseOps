### Rollback Scenario A — Deactivate Price Books (Soft Rollback, Preferred)

Use this if the price books need to be hidden from users without destroying historical data.

**Step A1 — Deactivate Evexia Price Book:**
```
sf data update record \
  --sobject Pricebook2 \
  --record-id 01sQl0000021f81IAA \
  --values "IsActive=false" \
  --target-org prod \
  --json
```
**Expected output:** `{ "result": { "id": "01sQl0000021f81IAA", "success": true } }`

**Step A2 — Deactivate Quest Price Book:**
```
sf data update record \

  --record-id 01sQl0000021f6PIAQ \
```

### Rollback Scenario B — Delete PricebookEntry Records (Hard Rollback)

> ⚠️ Only execute if no active Opportunity or Order line items reference these entries (confirmed in Pre-Deployment Checklist Step 2.7). Deletion of PricebookEntry records referenced by active Opportunities will fail with a dependency error.

**Step B1 — Retrieve all PricebookEntry IDs for Evexia:**
```
sf data query \
  --query "SELECT Id FROM PricebookEntry WHERE Pricebook2Id = '01sQl0000021f81IAA'" \
  --target-org prod \
  --json \
  --result-format csv > evexia_entries_to_delete.csv
```

**Step B2 — Retrieve all PricebookEntry IDs for Quest:**
```
sf data query \
  --query "SELECT Id FROM PricebookEntry WHERE Pricebook2Id = '01sQl0000021f6PIAQ'" \
  --target-org prod \
  --json \
  --result-format csv > quest_entries_to_delete.csv
```

**Step B3 — Bulk delete Evexia entries using CSV:**
```
sf data delete bulk \
  --sobject PricebookEntry \
  --file evexia_entries_to_delete.csv \
  --target-org prod \
  --json
```

**Step B4 — Bulk delete Quest entries using CSV:**
```
sf data delete bulk \
  --sobject PricebookEntry \
  --file quest_entries_to_delete.csv \
  --target-org prod \
  --json
```

**Verify deletion:**
```

  --query "SELECT COUNT(Id) remaining FROM PricebookEntry WHERE Pricebook2Id IN ('01sQl0000021f81IAA', '01sQl0000021f6PIAQ')" \
  --target-org prod \
  --json
```
**Expected output:** `remaining = 0`.

```
  --values "IsActive=false" \
  --target-org prod \
```
