### Test Case 3: Record-Triggered Flow — Field Sync on Opportunity Update (AccountId Set After Creation)

**Objective:** Verify that the Flow correctly populates Opportunity State and Main Practitioner fields when the Account is associated to an Opportunity after initial creation — addressing the auto-creation timing risk where Opportunities may be created before the Account relationship is established.

**Steps:**
1. In Sandbox, create a new Lab Consultation Opportunity with no Account linked (or with AccountId intentionally blank at creation).
2. Confirm that Opportunity-level State and Main Practitioner fields are blank after creation.
3. Edit the Opportunity and set the Account lookup to an Account with known, non-blank Billing State and Main Practitioner values.
4. Save the record.
5. Open the Opportunity and inspect the State and Main Practitioner fields.
6. Observe: fields should now reflect the linked Account's values.

**Result:** ✗ Blocked — Flow not yet built

**Notes:** This test case directly addresses the auto-creation timing risk identified in internal notes. If the Flow only fires on Create, this scenario will fail. The Flow must be configured to trigger on Create AND Update (when AccountId is set or changes).
