## Salesforce Problem

### Confirmed Facts

- The Salesforce Opportunity record `006Ql00000d7V5RIAU` is associated with patient Jeff Goodwin in CA.
- A Wellvi pharmacy order integration exists within the Salesforce org that sends order data to the Wellvi API.
- The Wellvi API is returning an `UnprocessableEntity` / `STATE_RESTRICTED` error for CA patients, blocking the submission.
- Per the reporter, CA is restricted only for Tirzepatide — TESA product orders to CA should be permissible.
- A character-count issue was also reported: the field was reduced below the previous threshold (to 103 characters including spaces), yet submission still fails — suggesting either Wellvi has tightened its character limit or the count methodology differs between Salesforce and Wellvi.
- The Wellvi integration was briefly suspected to be down (initial comment from David), but that was ruled out — the errors are functional, not connectivity-related.
- Danielle Cress (Technical Support Manager) was added to the Wellvi Slack channel by Jill Tager on 2026-05-11 to facilitate direct vendor communication.
- The issue was reported 2026-05-08 and as of 2026-05-11 remains unresolved (status: Waiting for support).

### Hypotheses

1. **Wellvi-side misconfiguration (most likely):** Wellvi's account settings for 10x Health are configured to restrict ALL CA orders (not just Tirzepatide), which is incorrect. The STATE_RESTRICTED error is being triggered by Wellvi's own routing rules, not by anything Salesforce is sending incorrectly. This requires Wellvi to correct their account/routing configuration.
2. **Salesforce sending incorrect product or state data in the API payload:** It is possible the integration is sending the wrong product identifier (e.g., a product code Wellvi maps to a restricted product), causing Wellvi to apply the wrong restriction. This would be an Engineering investigation into the Apex/flow logic that builds the API payload.
3. **Character-count discrepancy:** A field in Salesforce (likely a notes, instructions, or address field on the Opportunity or related order record) is being included in the API payload and Wellvi has a stricter or differently-counted character limit than previously assumed. This may be a separate issue from the STATE_RESTRICTED error or may be compounding it.
4. **Integration logic routing all CA orders through a restricted pathway:** The Salesforce integration code may be applying a blanket CA restriction that should only apply to Tirzepatide orders, rather than checking product type before routing.

### Likely Affected Metadata

- **Object:** `Opportunity` (record `006Ql00000d7V5RIAU`) — and likely a related Order or custom pharmacy order object
- **Integration component:** Apex class or Flow responsible for building and sending the Wellvi API callout (likely an `@future` or `Queueable` Apex class or named credential-based HTTP callout)
- **Named Credential / Custom Metadata / Custom Setting:** Wellvi API endpoint configuration, possibly including state restriction logic or routing rules stored in Custom Metadata Types or Custom Settings
- **Fields involved:** Patient state field (likely on the Opportunity or a related Contact/Account), product field (to identify TESA vs. Tirzepatide), and a notes/instructions field with a character limit
- **External system:** Wellvi pharmacy API (vendor-side configuration is a primary suspect)
