### Why This Should Fix It
Creating the `Supplement Inquiry` picklist field on the Case object and placing it in the Call Details section of the Case page layout directly addresses the gap the reporter identified. Customer Support agents will gain the ability to classify supplement-related cases using a standardized picklist, removing the blocker entirely.

### Risks
- **Picklist value formatting:** Two items in the original request ("Precision Weight Management Supplements 10X 5-MTHF") appear run together due to formatting; confirm with reporter whether these are one value or two separate values ("Precision Weight Management Supplements" + "10X 5-MTHF") before deploying.

- **No rollback complexity** — picklist fields can be deactivated or removed from layouts easily if needed.

### Rollback Plan
1. If the field causes unintended issues, remove `Supplement_Inquiry__c` from the Case page layout immediately (layout change — no data loss).
2. If needed, deactivate or delete the field from Setup → Object Manager → Case → Fields & Relationships (safe to delete if no data has been entered and no automations reference it).
