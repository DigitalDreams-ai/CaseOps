## Salesforce Problem

### Confirmed Facts
- The Salesforce org is `10xhealth.lightning.force.com`.
- The affected record is an Order object (`801Ql00000zolrtIAA`).
- Salesforce sends an outbound API call to Wellvi as part of the eSubmit action on the Order.
- Wellvi's API returns HTTP 400 with explicit errors stating City, PostalCode, AddressLine1, and StateProvince are missing — meaning these fields are null or empty in the JSON payload Salesforce sends.
- Sarah Phonthaphanh confirmed that shipping information is present on the Order record in Salesforce — meaning the data exists in the org but is not making it into the outbound payload.
- Danielle Cress initially suspected a Wellvi-side issue but subsequently confirmed the error is generated internally (Salesforce side) and is blocking submission; Wellvi never received an order.
- Only one record is currently reported as affected.
- The reporter's department is Operations; request type is Customer/Operations.
- System/Category is classified as "Salesforce – Opportunities – Opportunity Error" (though the record is an Order, not an Opportunity).

### Hypotheses
1. **Primary hypothesis — address field mapping bug:** The Salesforce integration component (Apex class, Flow, or REST callout) that builds the Wellvi API request body is reading from the wrong field(s) or the wrong object for shipping address data. The Order record has address data, but the integration is pulling from a field that is null on this record (e.g., billing address instead of shipping address, or a lookup to a related record that is not populated).
2. **Secondary hypothesis — data issue on this specific record:** The shipping address fields that feed the Wellvi payload (which may differ from the fields visible on the Order page layout) are actually blank on this record. The address visible on the UI may be stored on a different field or related object than what the integration reads.
3. **Tertiary hypothesis — recent data or record change:** Something on the Order record was modified prior to submission that cleared or nullified the address fields used by the integration (e.g., a field update via Flow, process builder, or manual edit).

### Likely Affected Metadata
- **Object:** `Order` (SObject — `801Ql00000zolrtIAA`)
- **Fields:** Shipping address fields on Order — likely `ShippingStreet`, `ShippingCity`, `ShippingState`, `ShippingPostalCode` — or a custom address field set used by the Wellvi integration
- **Integration component:** Apex class or Flow responsible for constructing and sending the Wellvi eSubmit HTTP callout (name unknown — Engineering must identify)
- **Named Credential / External Service:** Wellvi API endpoint configuration in Salesforce
- **Possible related objects:** Contact (`ShipToContact`), Account, or a custom address object linked to the Order
