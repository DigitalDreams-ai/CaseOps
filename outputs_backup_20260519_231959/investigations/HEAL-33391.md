### Hypotheses

1. **Primary hypothesis — Missing Permission Set or Profile permission:** Sarah's current Profile or Permission Set does not grant "Read" access to the Order object broadly enough, or the OWD for Orders is Private/restricted, preventing her from seeing records she does not own.
2. **Secondary hypothesis — Missing or inaccessible "All Orders" List View:** An "All Orders" list view may not exist, or it exists but is scoped to a specific group/role that excludes Sarah and her team. The user is defaulting to `__Recent` because no broader list view is available to them.
3. **Tertiary hypothesis — Role Hierarchy gap:** If OWD is Private and access is governed by Role Hierarchy, Sarah may not be positioned above the order processors in the hierarchy, preventing her from seeing their records via hierarchy-based visibility.
4. **Least likely — Field-level security:** Possible but unlikely to be the root cause given the nature of the complaint (can't see orders at all vs. can't see specific fields).

### Why This Should Fix It

Granting the appropriate Permission Set with Order object Read access, combined with a visible "All Orders" list view and (if needed) a Sharing Rule to overcome Private OWD, will allow Sarah and her team leads to see all orders regardless of record ownership. This directly addresses the root cause: insufficient object-level permissions and/or missing list view access.

### Risks

- **Over-permissioning:** If a broad Permission Set is assigned without scoping, users may gain access to more Order records or fields than intended. Scope the Permission Set carefully.
- **Sharing Rule breadth:** A poorly scoped Sharing Rule could expose all orders to too many users. Use a Public Group or Role-based criteria.
- **List View visibility:** If the "All Orders" list view is set to "All Internal Users," it may surface sensitive order data to users who should not have it. Validate sharing scope.
- **4 users vs. broader team:** Sarah's initial request said "only me" but follow-up expanded to 4. Confirm there are no other team members also needing this access before closing.

### Rollback Plan

- If a new Permission Set was deployed: remove the Permission Set assignment from the 4 users and delete or deactivate the Permission Set in Production.
- If a Sharing Rule was created: delete the Sharing Rule — records revert to OWD visibility.
- If a List View was created/shared: revert the list view sharing to "Just Me" or delete it.
- No code changes involved — all rollback steps are declarative config reversals.
