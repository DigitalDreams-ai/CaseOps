## Salesforce Problem

### Confirmed Facts

- **Three Lab Orders confirmed in Production:** `a47Ql000000JI2rIAG` (LAB-000228814, 03/02/2026), `a47Ql000000Lo9tIAC` (LAB-000232068, 04/07/2026), `a47Rh0000001zeEIAQ` (LAB-000058193, 05/27/2024). All linked to correct Opportunities.
- **Three Opportunities confirmed in Production:** `006Rh000009YUMfIAO`, `006Rh00000GVH6cIAH`, `006Ql00000YpadlIAB`. Each contains ONE OpportunityLineItem: the main PGT product (Product2 ID `01tRh000003qAcfIAE`, list price $1299).
- **Novogenia fulfilled all three orders with `#VRSF` product code** — confirmed by Courtney Marlin's review of Novogenia Portal; all three orders show `#VRSF - Report Print & Shipment Fee` as a fulfilled line item in the external Novogenia system.
- **Salesforce has NO corresponding record for the PGT Booklet supplement** on any of the three Opportunities. Specifically:
  - No OpportunityLineItem for the "Printed Report Books" product (Product2 ID `01tRh000006VKS1IAO`, exists in Salesforce but no ProductCode).
  - No custom `Lab_Order_Product__c` junction object (does not exist in the org).
  - No child Lab Order records or line items tracking the `#VRSF` product submission.
  - No ProductCode entry in Salesforce for `#VRSF` itself.
- **Product mismatch in Salesforce:** The "Printed Report Books" product (`01tRh000006VKS1IAO`) is the semantic equivalent of Novogenia's `#VRSF`, but it has no ProductCode set, no Pricebook entry visible, and no record linking it to these Opportunities.
- **Novogenia integration code not found in checked-in repo** — integration is Production-only or managed package; cannot be reviewed locally. Integration successfully calls out to Novogenia (orders fulfilled) but failed to create corresponding Salesforce records.
- **Nebula Logger available in Production** — actively used by Flows (confirmed in `Autolaunch_Create_Order_for_Fulfilled_By_Items.flow`). Execution logs for the affected Lab Order submissions exist but require browser UI access to retrieve detailed traces.
- **Historical logs partially inaccessible:** At least one order from 2024 is beyond the standard 30-entry debug log limit. EventLogFile API records exist but require manual download and parsing.

### Root Cause (High Confidence)

The Novogenia integration code includes logic to append `#VRSF` (PGT Booklet product) to outbound API payloads when certain conditions are met, **but the corresponding Salesforce record-creation step either:**

1. **Does not exist** — the integration calls out to Novogenia with `#VRSF` in the payload but has no code path to create an OpportunityLineItem or Lab Order product record in Salesforce.
2. **Exists but is broken or disabled** — a Flow action or Apex method to create the record fails silently, is wrapped in a try-catch that suppresses errors, or is conditionally disabled.
3. **Fails only under specific conditions** — the record-creation logic may have a bug or missing field mapping that causes it to fail for the `Printed Report Books` product specifically, while the Novogenia callout succeeds.

**Supporting evidence:**
- Novogenia Portal confirms `#VRSF` was in the API payload for all three orders (external source of truth).
- Salesforce has the "Printed Report Books" product (`01tRh000006VKS1IAO`) defined but it has no ProductCode, no Pricebook entries, and no records linked to these Opportunities.
- No custom data model exists for multi-product Lab Orders (`Lab_Order_Product__c` does not exist); Lab Orders support only a single `Product__c` field, suggesting they were not designed to track supplemental products.
- No Salesforce record whatsoever exists for the Booklet on any of the three Opportunities (not even a stub or failed transaction).
- Two of the three orders are from 2024; if this is a recent feature, the legacy integration may never have included the record-creation logic.

**Most likely scenario:** The integration was modified to send `#VRSF` to Novogenia (perhaps a recent change), but the corresponding Salesforce record-creation code either was not written or was added in one branch but not merged. Novogenia accepts the `#VRSF` product unconditionally, so the API succeeds; Salesforce never records the order because the code path doesn't exist or is broken.

### Likely Affected Metadata

- **Object: `Lab_Order__c`** — Custom object representing lab orders; the PGT Booklet supplement should exist as a child or related record here.
- **Object: `Opportunity`** — Standard Salesforce object; PGT Booklet orders should be associated via Opportunity Products or a related Lab Order.
- **Object: `OpportunityLineItem` / `Product2` / `Pricebook2`** — Product code `#VRSF - Report Print & Shipment Fee` should be mapped here; absence of a mapping could prevent record creation.
- **Apex classes or triggers** — Any Apex that handles outbound API calls to Novogenia and/or creates Lab Order records in Salesforce; likely the integration layer responsible for the submission.
- **Flow(s)** — Any active Record-Triggered or Scheduled Flows on `Lab_Order__c` or `Opportunity` that handle PGT Booklet product bundling or supplemental order creation.
- **Named Credentials / External Services / Custom Metadata** — Any integration configuration governing the Novogenia API call payload construction (e.g., `#VRSF` inclusion logic).
- **Debug Logs / Event Log Files** — Required to trace the original API call payload; inaccessible for 2024 orders via standard tooling.
