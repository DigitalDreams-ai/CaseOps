## Salesforce Problem

### Confirmed Facts

- The Salesforce org integrates with Wellvi via API to submit pharmacy orders.
- Wellvi enforces a hard limit of **125 characters (including spaces)** on the `RxClaim.Instructions` field in its API.
- The Salesforce Order record `801Ql00000zF7HPIA0` is the specific failing record.
- The error is originating from the Wellvi API response, not from within Salesforce itself — Salesforce is successfully calling Wellvi but Wellvi is rejecting the payload.
- The reporter states they already reduced the character count and the order is still failing, suggesting either: (a) the count is still above 125 including spaces, or (b) there is a caching/data issue where the old value is still being sent, or (c) a different field or concatenated value is inflating the instruction string.
- NDC: `bc178994-47c1-403a-aa30-3e33a0d9f840` was referenced by Sean Bingham as the source of the error trace.

### Hypotheses

1. **Primary hypothesis — Instructions field still exceeds 125 characters including spaces:** The user reduced the character count but may not be counting spaces. The Wellvi API counts spaces as characters and the value still exceeds 125.
2. **Secondary hypothesis — Concatenated/derived instructions value:** The Salesforce integration layer (Apex or Flow) may concatenate the user-entered instructions with other text (e.g., dosage, frequency, label text) before sending to Wellvi, resulting in a combined string that exceeds 125 characters even if the raw field looks short.
3. **Tertiary hypothesis — Stale/cached data being sent:** The record may be referencing a cached or previously saved value of the Instructions field that was not updated when the user shortened it. A hard save and re-fetch may be needed.
4. **No Salesforce-side character limit enforced:** There is likely no validation rule on the Order (or related) object limiting the Instructions field to 125 characters, allowing values above the Wellvi threshold to be saved and submitted.

### Likely Affected Metadata

- **Object:** `Order` (standard Salesforce object, `801Ql00000zF7HPIA0`)
- **Field:** The field on the Order object (or a related child/junction object) that maps to `RxClaim.Instructions` in the Wellvi API payload — API name unknown, likely a custom text or long text area field (e.g., `Instructions__c` or similar)
- **Integration layer:** Apex class or Flow that constructs the Wellvi API callout payload — this is where `RxClaim.Instructions` is populated and sent
- **Potential metadata fix:** A Salesforce Validation Rule on the Order object limiting the Instructions field to 125 characters to prevent submission of oversized values
- **Potential metadata fix (alternative):** An Apex-level truncation or guard in the callout class to enforce the 125-character limit before sending

## Solution Plan

### Classification

- **Immediate/record-level fix:** Support-resolvable — verify and correct the Instructions field value on the specific Order record.
- **Durable/platform fix:** Engineering escalation — add a validation rule or Apex guard to prevent this class of error from recurring.
- Escalation reason: The underlying gap (no character limit enforcement at the Salesforce layer) requires an Apex or validation rule change that Engineering owns.

### Proposed Change

**Step 1 (Immediate — Support/Operator):**
1. Open Order `801Ql00000zF7HPIA0` in Production.
2. Locate the Instructions field (the field that maps to `RxClaim.Instructions` in the Wellvi payload).
3. Count the current value's character length **including spaces** (use a tool like wordcounter.net or a spreadsheet `=LEN()` function).
4. If > 125 characters, reduce to ≤ 125 and save.
5. Resubmit the pharmacy order to Wellvi.

**Step 2 (Durable — Engineering):**
- Identify the Salesforce field API name that maps to `RxClaim.Instructions`.
- Add a Validation Rule on the Order object: `LEN(Instructions__c) > 125` → error "Instructions must be 125 characters or fewer (including spaces) to meet Wellvi pharmacy submission requirements."
- Alternatively or additionally, add an Apex-level guard in the Wellvi callout class to throw a descriptive exception before sending if the field exceeds 125 characters.

### Why This Should Fix It

The Wellvi API rejects payloads where `RxClaim.Instructions` exceeds 125 characters. Reducing the field value to ≤ 125 characters (including spaces) on the specific record will allow the API to accept the payload. The durable fix prevents recurrence by blocking oversized values at the Salesforce data-entry or code layer before they reach the Wellvi API.

### Risks

- **Clinical risk:** Truncating instructions may omit medically important information. Any shortening of pharmacy instructions should be reviewed by a clinical or operations lead before resubmission.
- **Concatenation risk:** If the integration layer appends additional text to the field value before sending to Wellvi, the raw field value must be shorter than 125 characters to account for the appended text. This must be confirmed before assuming a 125-character raw value is safe.
- **Scope of impact:** Other orders with similarly long Instructions values may also be failing silently or failing without tickets. A SOQL query on Order records submitted via Wellvi where instructions exceed 125 characters would identify additional affected records.

### Rollback Plan

- **Data fix:** If shortening the Instructions field causes issues, the original value can be restored manually. No metadata was changed.
- **Validation rule:** If the Validation Rule blocks legitimate clinical entries, it can be deactivated in Setup without a deployment.
