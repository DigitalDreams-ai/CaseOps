## Issue Understanding

### Observed Behavior

- Several hundred PAM cases (3/18–3/27/2026) for Shopify orders lack "Informed Consent 2026" recorded on the SF Account
- A fix released 3/27 resolved the issue for new MGT/GHT orders—recent orders now correctly populate Informed Consent on the Account
- Consent assignment appears to have failed only during the 3/18–3/27 window
- Removal of consent from SF after Chrono sync does not currently occur (historically absent as well)

### Expected Behavior

For Shopify MGT/GHT purchases:
1. Consent is assigned in Shopify
2. Synced to Salesforce (Account or Opportunity) on record creation
3. Forwarded to Chrono
4. Removed from Salesforce (per reported design intent)

### Acceptance Criteria

- Clarify which system (Shopify vs. Salesforce) is source of truth for Informed Consent
- Confirm intended workflow for consent assignment on account/opportunity creation
- Verify "Assign patient agreements" action is correct remediation for backfill
- Validate that all ~300+ affected cases receive consent via remediation
- Confirm post-fix behavior (consent properly syncs and is removed from SF after Chrono transfer)

### Unknowns

- Which Salesforce field/object holds Informed Consent (Account vs. Opportunity; field name)
- Specific flow/trigger responsible for consent assignment on Account/Opportunity creation
- Why consent removal from SF does not occur post-Chrono sync
- Exact scope of affected records (estimate ~300–hundreds, precise count unknown)
- Whether "Assign patient agreements" bulk action correctly maps Shopify consent to SF field
- Whether fix on 3/27 addressed root cause or merely re-triggered sync

## Salesforce Problem

### Confirmed Facts

- Informed Consent data exists in Shopify for affected orders (3/18–3/27)
- Consent is **not** present on corresponding SF Account records for affected window
- Post-3/27 fix, new orders correctly populate "Informed Consent 2026" on Account
- A bug fix or flow change was released 3/27 that resolved the population issue

### Hypotheses

- **H1:** Sync flow from Shopify → SF failed during 3/18–3/27; fix restored trigger/API call
- **H2:** Consent assignment field mapping changed on 3/27; earlier records used wrong field or no field
- **H3:** Account/Opportunity creation did not execute consent assignment flow during this window (prerequisite missing)
- **H4:** Consent removal from SF (post-Chrono) is design intent but not yet implemented (affects all records, not just 3/18–3/27 cohort)

### Likely Affected Metadata

- Account object: "Informed Consent 2026" field (or similar; exact API name needed)
- Flow or Apex on Account/Opportunity creation (trigger, handler, or sync process)
- Integration between Shopify and Salesforce (webhook, scheduled sync, or middleware)
- Chrono → Salesforce consent removal logic (if it exists)
