## 5. Rollback Plan

Rollback procedures are tiered by severity. Execute only the tier required for the specific failure observed.

⚠️ **Before executing any rollback:** Confirm whether any users have entered data into `Scheduled_By__c` since deployment (use the query in Step 2.8, repeated post-deploy). Field deletion (Tier 3) will permanently destroy all stored values after the Salesforce 15-day soft-delete window expires.

### Tier 1 Rollback — Remove Field from Layouts Only (Lowest Risk, No Data Loss)

**Use when:** Field is deployed but causes confusion, layout crowding, or is placed incorrectly. Field itself is fine; just hide it from users temporarily.

**What it does:** Removes `Scheduled_By__c` from all 6 page layouts. Field remains on the object and data is preserved. Users can no longer see or edit it from record UI.

**Steps:**
1. Revert all 6 layout files in your local project to their pre-HEAL-33116 state (restore from git)
2. Deploy the reverted layouts:

```
sf project deploy start --metadata "Layout:Opportunity-Operations Hormone Evaluation" --metadata "Layout:Opportunity-Operations Medical Consultation" --metadata "Layout:Opportunity-Retention Hormone Eval Opportunity Layout" --metadata "Layout:Opportunity-Retention Medical Consultation Opportunity Layout" --metadata "Layout:Opportunity-Sales - Retention Hormone Evaluation Page Layout" --metadata "Layout:Opportunity-Sales - Retention Medical Consultation" --target-org prod --json
```

**Verify rollback:**
1. Navigate to Setup → Object Manager → Opportunity → Page Layouts → open any of the 6 layouts
2. Confirm `Scheduled by` is in the "Fields not on Layout" palette, not placed on the layout
3. Navigate to record `006Ql00000bJEWbIAO` — confirm `Scheduled by` field is no longer visible

### Tier 3 Rollback — Delete the Field (Irreversible After 15 Days)

**Use when:** Feature is fully abandoned, the field caused a technical regression, or explicit stakeholder decision to remove it permanently.

⚠️ **WARNING:** This is destructive. Any data entered in `Scheduled_By__c` will be permanently deleted after the 15-day Salesforce soft-delete window. Do NOT execute this tier without explicit confirmation from Sean Bingham and stakeholder sign-off.

**Step 1 — Check for data before deleting:**

```
sf data query --query "SELECT COUNT() FROM Opportunity WHERE Scheduled_By__c != null" --target-org prod --json
```

If count > 0, export the data before deleting:

```
sf data query --query "SELECT Id, Name, Scheduled_By__c FROM Opportunity WHERE Scheduled_By__c != null" --target-org prod --result-format csv > scheduled_by_data_backup.csv
```

**Step 2 — Remove field from all layouts first (run Tier 1 rollback above)**

Fields cannot be deleted while referenced in page layouts. Complete Tier 1 before proceeding.

**Step 3 — Remove FLS from permission sets (run Tier 2 rollback above)**

**Step 4 — Delete the field via destructive change:**

Create a `destructiveChanges.xml` file with the following content:

```
<?xml version="1.0" encoding="UTF-8"?>
<Package xmlns="http://soap.sforce.com/2006/04/metadata">
    <types>
        <members>Opportunity.Scheduled_By__c</members>
        <name>CustomField</name>
    </types>
</Package>
```

Also create an empty `package.xml`:

```
<?xml version="1.0" encoding="UTF-8"?>
<Package xmlns="http://soap.sforce.com/2006/04/metadata">
    <version>59.0</version>
</Package>
```

Then deploy the destructive change:

```
sf project deploy start --destructive-changes-post destructiveChanges.xml --manifest package.xml --target-org prod --json
```

**Verify deletion:**

```
sf data query --query "SELECT Id, DeveloperName FROM FieldDefinition WHERE EntityDefinition.QualifiedApiName = 'Opportunity' AND DeveloperName = 'Scheduled_By__c'" --target-org prod --json
```

**Expected output:** `totalSize` = `0`

**Note:** Salesforce soft-deletes custom fields for 15 days. During this window, the field can be recovered via Setup → Object Manager → Opportunity → Fields & Relationships → Deleted Fields → Restore. After 15 days, deletion is permanent and irreversible.
