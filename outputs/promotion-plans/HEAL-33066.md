### Why No CLI Deployment Is Possible

- `sf project deploy start` deploys **metadata** (fields, flows, classes, objects, etc.) — not **data records**.
- Pricebook2, PricebookEntry, and Product2 are **data records stored in the org database**, not metadata components tracked in source control.
- Attempting `sf project deploy start --metadata "Pricebook2"` would fail — Pricebook2 is a standard Salesforce object, and its individual record instances are not deployable metadata artifacts.
- The correct tooling for data record promotion (if needed in the future) would be **Salesforce Data Loader** or **sf data import** using a CSV export from source org — but this is moot because the records already exist in Production.
