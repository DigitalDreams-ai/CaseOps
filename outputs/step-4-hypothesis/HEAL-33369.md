# Step 4 — Problem Hypothesis and Solution

**Issue:** HEAL-33369  
**Date:** 2026-05-20

---

## Problem Hypothesis

**Confirmed facts (from Step 3):**
- Opportunity Field History report accurately tracks Time to Schedule (TTS) metric
- Reporter needs to add Account-level billing state and main practitioner fields to the report for state and provider-level segmentation
- Current workaround: manual Excel cross-reference between Salesforce Opportunities and EHR appointment data (error-prone)
- Opportunity level has fields for billing state and main practitioner, but they are blank (data not pulling from Account)
- Report type is "Opportunity Field History"

**Symptoms (what the user reported):**
- Cannot display Account-level columns on Opportunity Field History report
- Opportunity-level fields exist but are unpopulated (no sync from Account)
- Requires manual Excel workaround
- Three reports affected (Lab Review highest priority)

**Root cause hypothesis:**
Opportunity Field History report type does not support direct parent Account references. Opportunity-level fields (billing_state, main_practitioner) exist but lack automation to sync values from the parent Account.

---

## Smallest Viable Fix

**What to fix:**
- **Metadata artifacts:** 
  1. Account billing state and main practitioner field definitions (verify they exist and their API names)
  2. Opportunity billing state and main practitioner field definitions (if they don't exist, create them as formula fields referencing Account parent)
  3. Data sync mechanism: Flow or Apex trigger to populate Opportunity fields from Account when Opportunity is created or Account is updated
  4. Opportunity Field History report definition (add new columns if fields don't exist, or activate them if hidden)

**Change scope:**
- Create formula fields on Opportunity (if needed) that reference Account parent
- OR create/enable flow automation to sync Account values to Opportunity fields
- Add exposed columns to the Opportunity Field History report

**Why it solves the problem:** 
With Account-level data accessible at the Opportunity level via formula fields or sync automation, the Opportunity Field History report can display billing state and main practitioner columns without requiring a custom report type or manual cross-reference.

## Risks and Constraints

**Implementation risks:**
- If using formula fields, potential performance impact if Account fields are not optimized for large queries
- If using flow-based sync, sync lag during bulk Account updates
- Custom formula fields may limit report type flexibility

**Constraints:**
- Cannot modify Opportunity Field History report type itself (platform limitation)
- Account fields must exist or be created before Opportunity sync can reference them
- Flow automation must respect Salesforce governor limits during bulk operations

**Mitigation:**
- Test formula field performance with large dataset in Sandbox
- Use scheduled flow or batch sync for bulk Account updates to avoid sync lag
- Document field naming convention for consistency across the three reports
