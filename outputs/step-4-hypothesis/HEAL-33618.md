# Step 4 — Problem Hypothesis: HEAL-33618

**Issue:** HEAL-33618  
**Date:** 2026-05-20

## Root Cause Hypothesis

Case object has IsEscalated checkbox and Escalation_Details__c field but NO timestamp capture. Round-robin flow routes Tier 1→Tier 2 but doesn't record WHEN escalation happened. Customer Support cannot report elapsed time from case open to escalation or escalation to closure.

## Smallest Viable Fix

1. **Create Case.Escalation_DateTime__c** (DateTime field)
2. **Modify Record_Trigger_Case_Assign_Round_Robin_Cases flow** — add action to populate timestamp when case escalates (when Dispo_Level_3__c = "Escalation")
3. **Add field to Case layout** (Case-Customer Experience, Escalations section)
4. **Set FLS** — match existing Escalation_Details__c permissions
5. **Create reports** showing elapsed time calculations

## Sandbox Validation Plan

1. Create test Case in Sandbox
2. Trigger escalation (set Dispo_Level_3__c = "Escalation")
3. Verify Escalation_DateTime__c populates
4. Create formula fields for elapsed time: (Escalation_DateTime__c - CreatedDate), (ClosedDate - Escalation_DateTime__c)
5. Build simple report showing both elapsed times

## Production Deploy

Gearset: custom field + flow modification + layout update + reports
