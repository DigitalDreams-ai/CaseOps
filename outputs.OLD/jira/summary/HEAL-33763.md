Sean, here are 3 permissions we would like to extend to the Tier 1 staff, which includes: Ray Guevarra Earth Alberio Genevieve Aguilar Marianne Sallo PERMISSION 1: REMOVE FILES FROM SALESFORCE OPPORTUNITIES Tier 1 regularly supports GT Opportunity workflows that require result regeneration. When prior result files are already attached to an Opportunity, the regeneration process is blocked and must be escalated to Tier 2 solely to remove the file — a task that adds no diagnostic value and consumes Tier 2 capacity unnecessarily. Approving this permission would eliminate a class of escalations driven entirely by a permissions gap, reduce average turnaround time for result delivery, and free Tier 2 bandwidth for higher-complexity issues. Risk is low. Salesforce's native Recycle Bin functionality ensures that any file removed in error can be fully recovered. PERMISSION 2: VIEW ACCOUNTS & OPPORTUNITIES OWNED BY ANDY HAAS Accounts created through the MyWellness API are systematically assigned to the user Andy Haas. Because Tier 1 lacks visibility into records owned by this user, agents cannot access or investigate these accounts without first requesting a Tier 2 ownership reassignment — introducing delay and adding unnecessary volume to the Tier 2 queue. Approving this permission would enable Tier 1 to directly access MyWellness API-originated accounts, reduce dependency on Tier 2 for routine lookups, and improve the team's ability to identify and flag duplicate accounts at first contact. Risk is low. This is a read-only visibility request. Tier 1 would have no ability to modify ownership, edit records, or alter opportunity data. PERMISSION 3: ADD ORDER NOTES TO CLOSED OPPORTUNITIES Tier 1 regularly handles inbound requests related to discounts, refunds, and order adjustments. Once an Opportunity is closed, Tier 1 is currently unable to add notes, forcing agents to route documentation through Tier 2 or leave interactions undocumented. Approving this permission would ensure timely and accurate documentation at the point of first contact, eliminate unnecessary Tier 2 involvement in a purely administrative task, and strengthen audit trails for financial and operational review. Risk is low. This permission is scoped to note creation only and does not grant the ability to reopen, edit, or modify the underlying Opportunity record or its financial data.
