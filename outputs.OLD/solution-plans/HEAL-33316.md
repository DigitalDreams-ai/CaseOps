### Rollback

Since no change has been made, there is nothing to roll back at this time.

When a fix is implemented, rollback procedure is:
- **Flow change:** Reactivate the previous Flow version in Setup > Flows (Salesforce retains all prior versions).
- **Validation rule change:** Re-enable or restore original criteria via Setup > Object Manager > Validation Rules.
- **FLS change:** Remove the permission from the profile/permission set via Setup > Profiles or Permission Sets.
- **All changes:** Document the pre-change state (screenshot or retrieved XML) before applying any fix so rollback is unambiguous.

## Why This Is the Minimal Viable Fix

**We are not proposing a configuration change because we have no confirmed root cause.** Any specific metadata change made now — editing a Flow, modifying a validation rule, adjusting FLS — would be speculation based on a one-line ticket. The risk of a speculative fix is higher than the risk of waiting 2–3 business days for the customer to respond.

The minimal viable fix at this stage is the smallest action that unblocks progress: a follow-up message and screenshot analysis. This costs zero risk and zero deployment effort. Larger alternatives — redesigning the pharmacy order Flow, restructuring permissions, or engaging Engineering — are not justified until the error message is known. We defer all configuration decisions to Step 4, where they can be scoped precisely to the confirmed failure point.
