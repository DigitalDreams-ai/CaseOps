# Solution Plan - HEAL-33150

## Problem Statement

Internal approval replies for Customer Experience cases are missing from Case Activity. The reported example includes a Salesforce thread token, but the Outlook reply path goes back to the agent and/or shared Outlook recipients rather than to Salesforce.

## Root Cause

The reply is bypassing Salesforce. Thread token matching only works after Salesforce receives the inbound email. Production examples prove Cx threading works when replies are delivered to routed mailboxes such as `support@10xhealthsystem.com` or `info@10xhealthsystem.com`.

## Smoking Gun

The exact action is `Case.SendEmail` on the `Case-Customer Experience` layout.

- Action label/type: `Email` / `SendEmail`
- Target object: `EmailMessage`
- Target parent field: `Parent`
- Sender field: editable `ValidatedFromAddress`
- Existing predefined field override: `ToIds = Case.ContactId`
- Missing predefined field override: no enforced `ValidatedFromAddress`
- Missing reply control: no Reply-To field on the action
- Affected outbound: `ValidatedFromAddress = ntorres@10xhealthsystem.com`
- Affected outbound: `EmailTemplateId = null`, `EmailRoutingAddressId = null`

This is the source of the failure. The internal approval path sends as the agent unless the user chooses a routed sender, so Outlook replies can bypass Salesforce.

## Proposed Fix

Engineering/Admin should make the internal approval email workflow route replies back through Salesforce.

Minimum viable path:

1. Update or replace the `Case.SendEmail` internal approval path.
2. Choose one supported routing fix:
   - Set Reply-To/From to an existing Salesforce-routed mailbox used for Case processing.
   - Configure the shared approval/refund mailbox to forward thread-tokened replies into Email-to-Case.
   - Replace the Outlook approval reply pattern with a Salesforce-native approval/comment workflow if the reply must be auditable on the Case.
3. Validate with an Outlook reply in Sandbox.

## What Not To Do

Do not treat the fix as simply creating a new Cx Email-to-Case routing address. That will not capture replies addressed only to an agent mailbox.

Do not rely on Bcc to the agent or per-user Email-to-Salesforce to capture future replies. Those mechanisms can log sent/outbound context, but they do not make Salesforce receive later Outlook replies.

## Validation

1. Create or use a Sandbox Customer Experience Case.
2. Send the same internal approval email currently used in Production.
3. Reply from Outlook and confirm whether the reply appears as `Incoming = true` on the original Case.
4. Apply the selected Reply-To/forwarding fix.
5. Repeat the reply test and verify the inbound `EmailMessage` attaches to the original Case.
6. Confirm existing customer replies to `support@10xhealthsystem.com` / `info@10xhealthsystem.com` still attach correctly.
