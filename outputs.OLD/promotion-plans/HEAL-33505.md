## 1. Summary

### What Changes in Production

**Scenario A — Field type is trackable (Text, Picklist, Number, Date, etc.):**

| Component | Object | Change Type | Details |
|---|---|---|---|
| Order Notes field | Opportunity | Metadata attribute update | `trackHistory` attribute set to `true` on the field definition |
| Opportunity Field History | Standard related list | No change required (already exists platform-wide) | Will begin populating entries for Order Notes upon first edit after enablement |

No new objects, flows, triggers, validation rules, permission sets, or page layouts are created unless the Field History related list is confirmed absent from relevant page layouts (secondary step if needed).

**Scenario B — Field type is Long Text Area:**

| Component | Object | Change Type | Details |
|---|---|---|---|
| `Opportunity_Notes_History__c` | New Custom Object | Create | Stores audit records for Order Notes changes |
| `Opportunity__c` (lookup) | `Opportunity_Notes_History__c` | New Field | Links history record to parent Opportunity |
| `OldValue__c` | `Opportunity_Notes_History__c` | New Field (Long Text Area) | Stores previous Order Notes value |
| `NewValue__c` | `Opportunity_Notes_History__c` | New Field (Long Text Area) | Stores new Order Notes value |
| `ChangedBy__c` | `Opportunity_Notes_History__c` | New Field (Lookup to User) | Records which user made the change |
| `ChangedDate__c` | `Opportunity_Notes_History__c` | New Field (DateTime) | Records when the change occurred |
| `OpportunityOrderNotesHistoryTrigger` | Opportunity | New Apex Trigger | After-update trigger comparing old/new Order Notes values |
| Opportunity Page Layout(s) | Opportunity | Layout update | Add `Opportunity_Notes_History__c` related list |

**NOTE:** Scenario B requires Sandbox build and full deployment pipeline. Scenario A may be applied directly via Setup UI in Production after Sandbox validation, without a Gearset/CLI deployment. Both are covered in Section 3.
