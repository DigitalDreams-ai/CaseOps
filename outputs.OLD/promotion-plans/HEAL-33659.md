### Rollback Condition

Initiate rollback if:
- Any post-deployment validation step in Section 4 fails AND the failure cannot be corrected by a targeted re-deploy
- An unintended side effect is observed on Case records or Customer Support agent workflows
- Sean Bingham or Noah Scott requests rollback

### Rollback Step R1 — Remove Field from Page Layout

If the layout deploy completed but needs to be reverted, retrieve the pre-deployment layout state. Since we do not have a pre-deployment snapshot, remove the field manually:

1. In Production Setup → Object Manager → Case → Page Layouts → Customer Experience
2. Drag `Supplement Inquiry` out of the "Call Details" section and into the palette (unplaced fields area).
3. Save the layout.

To deploy the revert via CLI (requires a pre-deployment backup of the layout XML — operator must have saved this during the pre-deployment checklist):

```
sf project deploy start \
  --metadata "Layout:Case-Customer Experience" \
  --source-dir ./backup_HEAL33659_prod \
  --target-org prod \
  --json
```

**Pass criteria:** `Supplement_Inquiry__c` no longer appears in any Case page layout section. Field is still in the system (unplaced) — data is not affected.

### Rollback Step R2 — Revoke FLS on Permission Sets

Re-deploy each permission set from the pre-deployment backup (operator must have saved this during pre-deployment checklist). For each permission set:

```
sf project deploy start \
  --metadata "PermissionSet:PermSetAPIName" \
  --source-dir ./backup_HEAL33659_prod \
  --target-org prod \
  --json
```

If no pre-deployment backup exists, revoke FLS manually:
1. In Production Setup → Permission Sets → [Customer Support Permission Set]
2. Object Settings → Case → Fields
3. Locate `Supplement_Inquiry__c`
4. Uncheck Read Access and Edit Access
5. Save

Repeat for each affected permission set.

**Pass criteria:** Re-run Section 4d query — `PermissionsRead: false`, `PermissionsEdit: false` for all Customer Support permission sets.

### Rollback Step R3 — Delete the Field (Only If No Data Entered)

⚠️ ONLY execute if:
- The field was deployed in the current deployment session (not pre-existing)
- No Case records have a non-null value in `Supplement_Inquiry__c` (verify first with query below)
- Rollback has been explicitly authorized by Sean Bingham

First, confirm no data exists in the field:

```

  --query "SELECT COUNT() FROM Case WHERE Supplement_Inquiry__c != null" \
  --target-org prod \
  --json
```

**If count = 0 (safe to delete):** Delete the field via Setup → Object Manager → Case → Fields & Relationships → `Supplement_Inquiry__c` → Delete. Confirm deletion in the UI. CLI metadata delete:

```

  --metadata "CustomField:Case.Supplement_Inquiry__c" \
  --target-org prod \
```

**Expected output:**
