## 1. Summary

### What Will Change in Production

This is a user-level access change only. No metadata components are being created, modified, or deleted. No deployable metadata artifact (package.xml, change set, scratch org source) exists or is required for this change type.

| Component | Type | Change | Scope |
|---|---|---|---|
| Marianne Sallo — User record | User (non-deployable) | Permission Set assignment added | Single user only |
| Target Permission Set (name TBC — pending B-6 resolution) | Permission Set | No change to the component itself — only the assignment to the user | Additive only |
| FLS on specific fields (Fix B — conditional only) | Field-Level Security within existing Permission Set | Edit FLS added for named field(s) only | Only if error message from screenshots explicitly names a field; not assumed |

### What Will NOT Change

- No Profile modifications
- No Page Layout modifications
- No Flow activations or deactivations
- No Apex class changes
- No new Permission Sets created
- No Validation Rules touched
- No Custom Objects or Custom Fields modified
- No metadata deployed via CLI (Permission Set assignments are UI-applied in Production Setup; they are not deployable via `sf project deploy` in a way that is safe for a single-user targeted change without risk of overwriting org-wide Permission Set assignments)

### Deployment Method Note

Permission Set **assignments** (user-to-Permission Set mapping) are not retrievable or deployable as standard metadata in a way that safely targets a single user. Deploying a Permission Set definition via CLI would overwrite the component, not add an assignment. The correct and safe mechanism for this change is **manual Setup UI action in Production**, performed by Sean or a designated admin, following Sandbox validation. This is consistent with the solution plan's specified approach.

If Fix B (FLS on an existing Permission Set) is required, that Permission Set metadata CAN be retrieved from Sandbox and deployed to Production via CLI. CLI commands for that conditional path are included in Section 3.
