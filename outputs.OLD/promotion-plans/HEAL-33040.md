## 5. Rollback Plan

> Rollback returns Production to the pre-fix broken-but-stable state. The underlying bug (no sync) was already present before this deployment. Rolling back does not make things worse than they were before — it only removes the fix and returns sync to its prior non-functional state.

> Shopify customer records created during the fix window (between deploy and rollback) must be reviewed — confirm with Engineering whether they need to be removed from Shopify.

### Trigger for Rollback

Initiate rollback if any of the following occur after deployment:
- Apex tests fail on deploy (rollback before re-trigger steps)
- Existing Shopify-synced accounts show changed or missing `Shopify_Customer_ID__c` values (regression)
- Duplicate Shopify customer records are created for previously synced accounts
- Sync fires on Account record types or entry points it was not previously firing on (unexpected scope expansion)
