## 5. Rollback Plan

> **⚠️ No metadata was deployed; there is nothing to "undeploy."**
> Rollback for this ticket means reversing specific **data record changes** made in Production.
> All rollback steps are record-level operations in Salesforce.

### 5.1 Rollback — Deactivate a Specific Product (if activation was incorrect)

**Trigger condition:** A product from the 59-record activation list is found to have been activated incorrectly and must be returned to inactive.

**CLI command (replace `<ProductId>` with actual record ID from `InactivePricebookProducts.xlsx`):**
```
sf data update record \
  --sobject Product2 \
  --record-id <ProductId> \
  --values "IsActive=false" \
  --target-org prod \
  --json
```

**What it does:** Sets `IsActive = false` on the specified `Product2` record.

**Expected output:** `"success": true` in JSON response.

**Repeat for each product that requires deactivation.** Reference `InactivePricebookProducts.xlsx` for the full list of IDs.

**Verify rollback:**
```

  --query "SELECT Id, Name, IsActive FROM Product2 WHERE Id = '<ProductId>'" \
  --target-org prod \
  --json
```
**Expected:** `"IsActive": false`
