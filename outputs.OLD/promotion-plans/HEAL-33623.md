A link resend does not destructively alter existing records. The broken link state was already the current Production state before the resend. **There is no rollback required or possible for a resend — the pre-resend state was already broken.**

However, if the resend action creates an unintended side effect (e.g., a duplicate consent form record is created, or the resend triggers an unwanted automated process), use the pre-action snapshot to identify the delta:

**Step 1: Review the pre-action snapshot to identify unexpected changes**

```

  --query "SELECT Id, Name, Status__c, Link__c, CreatedDate, LastModifiedDate FROM ConsentForm__c WHERE AccountId = '001Rh00000GWgxNIAT' ORDER BY CreatedDate DESC LIMIT 10" \
  --target-org prod \
  --json
```

Compare output against `HEAL-33623-pre-resend-snapshot.json`. Identify any records that exist now but did not exist before the resend.
