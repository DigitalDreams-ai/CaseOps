# Internal Notes — HEAL-33765

**INTERNAL ONLY — DO NOT POST TO JIRA**

---

## Issue Summary

Global service members (3 users transitioning to Pre-charting role) cannot update the Main Practitioner field on Patient Account records.

## Steps to Reproduce

1. Log in as one of the Global service members (Pre-charting): adelavega@10xhealthsystem.com, cfernandez@10xhealthsystem.com, or jgallano@10xhealthsystem.com
2. Navigate to a Patient Account record (e.g., Account ID 001Rh000001CVZ8IAO)
3. Attempt to edit the "Main Practitioner" field on the Account

## Solution

Root cause: [Lab_Providers_C_R_E](sf://0PS5a000000GZQYGA4) permission set was missing FLS entry for [Account.Main_Practitioner__c](sf://00N0b00000B34Fc). Salesforce FLS logic requires ALL assigned permission sets to grant field access; missing FLS entry = silent denial.

Fix: Added FLS entry `<editable>true</editable><field>Account.Main_Practitioner__c</field><readable>true</readable>` to Lab_Providers_C_R_E permission set.

Affected file: `force-app/main/default/permissionsets/Lab_Providers_C_R_E.permissionset-meta.xml`

## Confirmation

Deployed to Sandbox 10xhealth-sean and validated field access for all three users. All users confirmed read/write access to Main Practitioner field in Sandbox. Ready for Production deployment via Gearset.
