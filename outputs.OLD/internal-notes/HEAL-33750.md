# HEAL-33750: Internal Notes

## Root Cause

[Permission Set: Wellness_Team](sf://permission-set/Wellness_Team) lacked Field-Level Security read/edit permission on [Field: OrderItem.Tracking_Number__c](sf://field/OrderItem/Tracking_Number__c), hiding the field from users despite it being present on the page layout.

## Decision

Support-resolved. No Engineering escalation needed.

## Actions Taken

1. Added FLS read/edit permission for [Field: OrderItem.Tracking_Number__c](sf://field/OrderItem/Tracking_Number__c) to [Permission Set: Wellness_Team](sf://permission-set/Wellness_Team)
2. Tested in Sandbox: Wellness Concierge users can now view and edit the Tracking_Number__c field on OrderItem records
3. Validation: Acceptance criteria met — team members have visibility into Wellvi tracking information

## Production Deployment State

Sandbox fix is validated and ready. Production metadata deployment required via Gearset to apply the FLS permission change to [Permission Set: Wellness_Team](sf://permission-set/Wellness_Team) in Production.
