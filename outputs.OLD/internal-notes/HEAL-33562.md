## Root Cause (Confirmed)

Two Salesforce flows auto-submit Opportunities for approval WITHOUT specifying the `submittedBy` parameter:
1. `Record_Trigger_Opportunity_Shopify_Automations`
2. `Record_Trigger_Opportunity_Superhuman_Protocol_Submit_For_Approval`

**Why Tyler cannot recall:**
- Flows submit without `submittedBy` parameter → Salesforce defaults submitter to system/automation user
- Salesforce Approval Process only allows the ORIGINAL SUBMITTER to recall
- Tyler (owner) ≠ System user (submitter) → Recall button hidden from Tyler

**Why Order Processing can recall:**
- Order Processing likely submits directly via UI, making themselves the submitter
- They can recall because they are the submitter

## Solution Applied

Modified both flows to add `submittedBy=$Record.OwnerId` parameter to submit actions. This designates the record owner as the submitter, enabling recall.

**Files Changed:**
1. `force-app/main/default/flows/Record_Trigger_Opportunity_Shopify_Automations.flow-meta.xml`
   - Submit_for_Approval action: Added submittedBy parameter (lines 622–627)

2. `force-app/main/default/flows/Record_Trigger_Opportunity_Superhuman_Protocol_Submit_For_Approval.flow-meta.xml`
   - Send_to_Approval action: Added submittedBy parameter (lines 11–15)
