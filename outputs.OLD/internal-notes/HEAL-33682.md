## Root Cause

When the original PGT Lab Orders were submitted to Novogenia via API, the payload included product code `#VRSF` (Report Print & Shipment Fee), triggering Novogenia to produce and ship a physical PGT Booklet. However, no corresponding Opportunity or Lab Order record exists in Salesforce for the booklet fulfillment — indicating the API call included the booklet product code without a matching SF order record being created (or the record was never created/was deleted). This is an upstream data integrity gap: SF did not persist or never generated the booklet order record at submission time. Confirmed by Courtney Marlin via Novogenia portal review. Affects at least 3 Lab Orders, 2 of which date to 2024, placing them outside the 30-entry log retention window available to Support.

## Decision

**Escalate to Engineering**

Confidence: High. Support cannot access API submission logs beyond the last 30 entries. The 3 affected Lab Orders (a47Ql000000JI2rIAG, a47Ql000000Lo9tIAC, a47Rh0000001zeEIAQ) require log-level investigation to determine whether: (a) the API call was malformed and included `#VRSF` in error, (b) a trigger/automation failed to create the SF booklet order record after submission, or (c) records were created and subsequently deleted. This requires Engineering-level log access.
