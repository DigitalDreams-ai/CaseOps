# HEAL-33889 — Internal Notes

## Root Cause

ShipStation OAuth token for SalesforceCore connector (SellerID:4177129, StoreID:793381) expired or was revoked on the external ShipStation platform between 2026-05-28 08:48 PM and error report time.

## Decision

Support coordination required. No Salesforce code or configuration changes needed. Customer must complete OAuth reauthorization in ShipStation admin console.

## Implementation

1. Provide customer with reauth procedure:
   - Log into ShipStation admin
   - Navigate to Salesforce store connector (SellerID:4177129, StoreID:793381)
   - Click "Reconnect" or "Reauthorize"
   - Complete OAuth flow
   - Confirm with test order creation

2. Verify sync resumes after reauth
3. Escalate if customer cannot access ShipStation admin or sync fails after reauth

## Sandbox Verification

Sandbox testing completed and passed. Salesforce Order triggers (OrderTrigger, OrderProductTrigger), MetadataTriggerHandler framework, and ShipStation Permission Sets confirmed active and functioning. No metadata deploy required.

## Risks and Escalation Criteria

- **Escalate to Engineering** if:
  - Sync continues to fail after customer completes ShipStation OAuth reauth
  - Named Credential storage or token refresh mechanism in Salesforce is misconfigured
  - Remote Site Settings or HTTP callout configuration is blocking valid OAuth tokens
  
- **Escalate to Account Manager** if customer cannot access ShipStation admin console

- **Escalate to ShipStation Support** if reauth succeeds but ShipStation-side token validation still fails (account compromise or platform-level issue)

## Production vs Sandbox

- Sandbox: Integration verified working
- Production: Customer OAuth reauth required, then test with new order creation
