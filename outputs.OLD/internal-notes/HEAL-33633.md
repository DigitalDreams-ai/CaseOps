# Internal Notes — HEAL-33633

**Issue:** Wellvi Submission error Joshua Brown  
**Date:** 2026-05-26  
**Status:** Support-Resolved (Sandbox validated)  
**Root Cause:** Wellvi/TenX API contract mismatch in CMT JSON path configuration  
**Disposition:** Metadata configuration fix deployed to Sandbox

---

## Summary

Earlier investigation identified a payload contract mismatch: the Wellvi/TenX endpoint requires address fields as root-level PascalCase properties (`City`, `PostalCode`, `AddressLine1`, `StateProvince`), but the live CMT mappings were sending them under nested lowercase paths (`$.shippingAddress.city`, etc.).

This is **not** a missing field or data entry issue. Address data IS populated on the Order and IS visible in the UI. The failure is in how the CMT JSON paths serialize the address fields into the API request body.

**Resolution:** Updated four `API_Parameter__mdt` records in Sandbox to map to the correct JSON paths at the root level. No code changes required.

---

## Root Cause Details

**Problem:** CMT mappings use lowercase nested paths:
- `Shipping_City__c` → `$.shippingAddress.city`
- `Shipping_Street__c` → `$.shippingAddress.addressLine1`
- `Shipping_State__c` → `$.shippingAddress.stateProvince`
- `Shipping_Zip_Code__c` → `$.shippingAddress.postalCode`

**API Contract:** Wellvi/TenX expects:
- `City`, `PostalCode`, `AddressLine1`, `StateProvince` at the root of the request body

**Result:** When the Order is submitted, the address fields are either missing or in the wrong location, triggering the HTTP 400 validation error from Wellvi.
