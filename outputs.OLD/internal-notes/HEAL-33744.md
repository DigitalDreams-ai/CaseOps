Jon Jasper Garing lacks CRUD:Create permission on Patient_Agreement__c object. The screenflow Screenflow_Send_Patient_Agreement_Forms runs in SystemModeWithoutSharing (bypassing user perms at flow level) but delegates record creation to subflow Autolaunch_Send_Patient_Agreements_to_Patient. The subflow executes DML CREATE on Patient_Agreement__c without system-mode overrides. Without permission set assignment, CREATE silently fails — user sees no record on Account related list and reports agreement "not showing up."
