## Root Cause
Four users on the Order Processing team (Sarah Phonthaphanh, Kaci Krupnik, Sarah Courchia, Kristhal Valdez) lack object-level Read access to the Order object. The "All Orders" list view is not visible to them due to missing permission set assignment.

## Decision
Support-Resolvable (High confidence — 92%)
This is a standard permissions request that can be resolved by auditing Production, validating in Sandbox, and assigning the appropriate permission set to the four named users.

## Actions Needed

1. **Audit Production Order permission sets**
   - Identify existing Order-related permission sets (e.g., "Order Processor Lead", "Order Manager", or similar)
   - Document which permission sets grant Read access to the Order object
   - Confirm which one(s) would be appropriate for an order processing team

2. **Validate in Sandbox**
   - Import the identified permission set(s) to Sandbox
   - Verify that the permission set grants Read access to the Order object
   - Confirm that users assigned to it can view the "All Orders" list view

3. **Deploy to Production**
   - Assign the validated permission set to: Sarah Phonthaphanh, Kaci Krupnik, Sarah Courchia, Kristhal Valdez
   - Ensure assignment is Read-only access (not Edit)

4. **Confirm with requestor**
   - Ask Sarah to verify that all four team members can now see the "All Orders" list view
   - Document confirmation and mark issue as resolved

## Red Flags & Risk Mitigation

- **Production audit not yet completed:** Must identify the correct permission set before making assignments. Do not guess or assign broadly.
- **Limited confirmation from affected users:** Only Sarah submitted the request; three of four users have not individually confirmed. Treat manager assertion as sufficient given operational context, but note dependency on Sarah's verification step.
- **Inadvertent Edit access risk:** Ensure permission set grants Read-only access to Order object, not Edit. Inadvertent Edit access poses data integrity risk for order processing.
