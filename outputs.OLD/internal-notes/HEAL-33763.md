## Root Cause

Tier_1 PermissionSet (ID: 0PSRh00000025ObOAI, label: "Tier 1") is missing three permission configurations in Production:

1. **Attachment delete permission** — objectPermission for Attachment object lacks allowDelete=true
2. **Account/Opportunity record visibility for Andy Haas records** — objectPermissions missing viewAllRecords=true flag (currently viewAllRecords=false)
3. **Opportunity.Notes field creation** — fieldPermission for Opportunity.Notes missing or editable=false

Each permission gap confirmed via SOQL query (Tier_1 PS confirmed to exist) and inference from 30+ similar Tier support permission sets reviewed in HEAL-33562 metadata dump.

## Fix Summary

**Metadata:** PermissionSet.Tier_1

**Three elements to add/enable:**
- Attachment object: allowDelete=true (all other flags: false/default)
- Account object: viewAllRecords=true (grants read-only access to all Account records)
- Opportunity object: viewAllRecords=true (grants read-only access to all Opportunity records, including Andy Haas-owned)
- Opportunity.Notes field: readable=true, editable=true

XML templates prepared and ready in source: `force-app/main/default/permissionsets/Tier_1.permissionset-meta.xml`

**Why this works:**
- Attachment.allowDelete=true unblocks file removal from Opportunity records (Recycle Bin provides recovery safeguard)
- Account/Opportunity.viewAllRecords=true eliminates sharing gap; Tier 1 users can then see all Accounts/Opportunities, including those owned by Andy Haas
- Opportunity.Notes fieldPermission enables note creation on closed Opportunities (no validation rule blocks this; field permission alone is sufficient)

- Mitigation: Confirm with Danielle Cress before Production deploy. If scope must be narrower, escalate to create sharing rule instead (more complex, requires ongoing maintenance)
