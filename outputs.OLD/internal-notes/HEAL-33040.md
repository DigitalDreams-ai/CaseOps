## Root Cause

**CRITICAL: SUBFLOW HAS FATAL DEFECTS**

Two sync mechanisms exist but both fail:

**1. Scheduled Flow (DISABLED)**
- Scheduled_Flow_Sync_Accounts_to_External_Systems (ID: 301Rh00000MdqwZIAR)
- Status: Obsolete (line 85 of flow XML)
- Schedule: Daily 18:00:00 UTC
- Calls subflow: Autolaunch_Account_Sync_To_External_Systems
- **Impact:** Never runs, can't catch up on missed accounts

**2. Record Trigger (ACTIVE but BROKEN)**
- Record_Trigger_Account_Account_Sync_To_External_Systems
- Status: Active, fires on Account creation/update
- Calls same subflow: Autolaunch_Account_Sync_To_External_Systems
- **Impact:** Fires on every new account but subflow fails silently
- **Result:** New accounts never synced, scope grows 10-20/day

**Subflow Critical Bugs (tested 2026-05-22):**
- Line 136: eventId assigned NULL (no value element in XML)
- Patient_Event_API receives null eventId, likely rejects request
- Patient_Event_API returns ZERO output (no Shopify customer ID returned)
- No Account update step (even if API succeeded, sync can't persist)
- No error handling (failures are silent, flow logs success anyway)

**Manual test result:**
- Account: 0010b00002LQU4xAAH (Jason Toff)
- Subflow ran without errors but Account.Shopify_Customer_ID__c remains null
- API action returned no output, no error messages

**Why scope expanding:** Record trigger fires on creation but subflow fails silently. New accounts created daily → sync attempted but fails → scope grows by 10-20 accounts/day.

## Decision

**ESCALATE TO ENGINEERING — CRITICAL SUBFLOW BUGS**

Confidence: Very High. Root cause identified and confirmed via manual testing: Subflow has three fatal defects preventing Shopify sync.

4. Re-activate `Scheduled_Flow_Sync_Accounts_to_External_Systems` (change Obsolete → Active) as backup for future catch-ups

**Do NOT re-activate scheduled flow until subflow is fixed** — it will fail silently like the record trigger.
