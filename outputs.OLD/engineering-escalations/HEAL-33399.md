**Problem:** Every CX Case in Salesforce is throwing an error when a user attempts to close the case (change status to Closed). This is a 100% failure rate — no cases can be closed. **Potential Fix:** Identify the active Flow(s) on the Case object that fire on close or status change → locate the broken element (likely a null reference, missing required field input, deactivated action, or broken subflow) → correct the element in Sandbox → deploy to Production.

## Root Cause

Unknown — not yet confirmed. Most likely a Flow on the Case object that triggers on close/status change contains a broken element. Possible causes include: a null record reference, a required Flow input that is no longer populated, a called action or subflow that has been deactivated or modified incompatibly, or a validation/decision element referencing a field that no longer exists or has changed. The error screenshot attached to Jira should be the first artifact reviewed — the error message text will likely identify the Flow name and failing element directly.

## Problem Location (Step 6)

**Problem Type:** Component (Flow automation)

**Artifact:** Flow (Record-Triggered or Screen Flow) on Case object with entry condition on status change to "Closed"

**Location:** Production > Setup > Flows — search for active Flows on Case object where trigger/entry condition = status field change to "Closed"

**Failure Point:** Flow element is throwing error at execution time during case close. Error is captured in Jira screenshot (HEAL-33399 attachment). Most likely: null reference, missing required input on called action/subflow, field reference to deleted/renamed field, or deactivated external action.

## Affected Component

- **Metadata/code component:** Case object — Record-Triggered Flow or Screen Flow on status change
- **Element/rule/process, if applicable:** The specific Flow element that fails when case is closed (identify from error message in attached screenshot)

## Potential Fix

The Flow on Case that executes on close is failing. Fix:
1. Locate the case-close Flow in Setup > Flows (use error message from screenshot to identify Flow API name and failing element).
2. Audit the failing element: check for null references, missing required inputs on called actions/subflows, references to deleted/renamed fields, deactivated external actions, or incompatible action updates.
3. Correct the element (fix reference, add missing input, reactivate action, or update to current API).
4. For emergency unblock: if fix is complex, consider temporary Flow deactivation to restore case-close capability (only if Flow is non-essential to core close logic).
5. Validate in Sandbox (clone affected CX case, test close action, confirm Flow completes without error).
6. Deploy to Production via Gearset or changeset.
