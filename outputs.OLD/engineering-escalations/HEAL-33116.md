# Engineering Handoff

## Engineering Message

**Issue:** HEAL-33116 — Calendly automation to auto-populate `Scheduled_By__c` = "Self-Scheduled" on Consultation Opportunity
**Problem:** A new picklist field `Scheduled_By__c` has been added to the Consultation Opportunity object in Salesforce. When a Calendly booking occurs and syncs to Salesforce as a scheduled Opportunity, there is no automation to set this field to "Self-Scheduled" by default. Currently the field will always be blank on Calendly-originated records unless manually updated.
**Potential Fix:** Add automation (Flow, Apex trigger, or Calendly webhook handler — whichever owns the Calendly-to-SF sync) to set `Scheduled_By__c` = "Self-Scheduled" when a Consultation Opportunity is created or updated via a Calendly booking event. The field must remain editable post-creation so schedulers can override it manually.

---

## Steps to Reproduce

**Affected record IDs:** Example Opportunity: `006Ql00000bJEWbIAO`

**Steps:**
1. Log in as an Operations user with a permission set that includes Calendly scheduling access.
2. Trigger a Calendly booking for a consultation appointment (or simulate the Calendly webhook/sync event that creates/updates the Salesforce Opportunity).
3. Navigate to the resulting Consultation Opportunity record in Salesforce.
4. Observe: `Scheduled_By__c` (label: "Scheduled by") field is blank.

**Expected behavior:** `Scheduled_By__c` should be automatically set to "Self-Scheduled" when the Opportunity is created or its status is set to scheduled via a Calendly sync. The field should remain editable so a scheduler can manually change it to any other picklist value (e.g., a specific scheduler name) if manual intervention occurred.

No automation exists between the Calendly booking sync and the new `Scheduled_By__c` field. The field was created by Support/Admin but the defaulting logic requires knowledge of the Calendly integration layer (webhook, middleware, or existing Flow/Apex) that is outside Support scope.
