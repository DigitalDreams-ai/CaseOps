### 2026-04-17T20:11:18.860-0400 - Sean Bingham (internal/unknown)

Findings: The flow Opportunity_to_Order sets Order.EffectiveDate from Opportunity.CloseDate. That is why refill Orders keep showing the original close date instead of the day the Order is created. We need to update Opportunity_to_Order so that on Order creation, Order.EffectiveDate is set from $Flow.CurrentDate, not from CloseDate. Reproduce: Open a Closed Won Opportunity with a past CloseDate (example: 006Rh00000QDlVBIA1). Create a new Order from that Opportunity (same refill path you use for refill POs). On the Order, check EffectiveDate: in production it matches Opportunity.CloseDate, not today (Cre8 will mirror that if it uses the same field).
