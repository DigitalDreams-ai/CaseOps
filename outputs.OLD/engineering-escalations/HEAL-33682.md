# Engineering Handoff

## Engineering Message

**Issue:** HEAL-33682 — Orphan PGT Booklets: No Opportunity or Lab Order record in Salesforce
**Problem:** Novogenia received API calls for PGT Lab Orders that included product code `#VRSF` (Report Print & Shipment Fee), causing physical PGT Booklets to be printed and shipped. No corresponding Opportunity or Lab Order record exists in Salesforce for the booklet component of these orders. The booklet fulfillment is "orphaned" — it happened in Novogenia but SF has no record of it.
**Potential Fix:** Identify the Apex trigger, Flow, or integration process responsible for creating SF booklet order records when `#VRSF` is included in a Lab Order API submission. Determine why record creation did not fire or persist for these 3 orders. If a systemic bug is found, patch the automation and assess whether historical orphan orders need backfill.

## Root Cause

Unconfirmed pending log review. Leading hypothesis: the Salesforce automation (trigger or Flow) responsible for creating a booklet order record upon detection of `#VRSF` in the Novogenia API payload either (a) failed silently, (b) was not in place at the time of original submission (2024 orders), or (c) records were created and subsequently deleted. A secondary hypothesis is that `#VRSF` was included in the API payload erroneously due to a product code mapping error, and no automation was ever intended to fire — meaning the gap is in Novogenia-side validation rather than SF record creation.

## Problem Location (Step 6)

**Problem Type:** Component (automation)

**Artifact:** Apex trigger or Flow on Lab_Order__c object that processes Novogenia API submissions and creates booklet order records when `#VRSF` product code is present

**Location:** Production > Setup > Object Manager > Lab_Order__c > Triggers (if Apex) or Flows (if Flow); Novogenia API integration handler

**Failure Point:** When Lab Order API submission includes `#VRSF` product code, the SF automation that should create a booklet order record either (1) failed silently, (2) was not deployed at time of original submission, or (3) contains logic that excludes this scenario.

## Affected Component

- **Metadata/code component:** Novogenia Lab Order API integration layer; Apex trigger or Flow on `Lab_Order__c` object
- **Element/rule/process, if applicable:** The automation step that processes `#VRSF` product codes and creates child/linked SF booklet order records

## Potential Fix

1. Pull API submission logs for the 3 Lab Order IDs to confirm `#VRSF` payload inclusion and any SF automation execution trace.
2. If automation bug confirmed: patch the trigger/Flow to correctly create booklet order records on `#VRSF` inclusion; validate in Sandbox against a test Lab Order with `#VRSF`.
