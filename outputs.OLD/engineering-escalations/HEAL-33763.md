# Engineering Handoff — HEAL-33763

## Engineering Message

**Issue:** HEAL-33763 — Giving Tier 1 Tech Support more SF access

**Problem:** Tier 1 support staff lack three permissions in Salesforce: ability to delete attachments from Opportunities, visibility into Andy Haas-owned Accounts/Opportunities, and ability to add notes to closed Opportunities. Each gap forces escalation to Tier 2 for administrative tasks.

**Potential Fix:** Add three configuration changes to Tier_1 PermissionSet:
1. Grant Attachment delete permission (objectPermission with allowDelete=true)
2. Grant read visibility for Andy Haas-owned records (objectPermission with viewAllRecords=true for Account and Opportunity)
3. Grant Opportunity.Notes field edit/read permission (fieldPermission with editable=true, readable=true)

## Root Cause

Tier_1 PermissionSet (API name: "Tier_1", ID: 0PSRh00000025ObOAI in Production) is configured with insufficient permissions:
- Attachment object permission does not include delete capability
- Account and Opportunity object permissions lack viewAllRecords flag (blocking visibility of Andy Haas-owned records)
- Opportunity.Notes field permission is missing or marked non-editable

These gaps are intentional architectural decisions but create unnecessary Tier 1 → Tier 2 escalations for routine administrative tasks that add no diagnostic value.

## Potential Fix

Add three configuration changes to the Tier_1 PermissionSet metadata:

### 1. Attachment Delete Permission
```xml
<objectPermissions>
  <allowCreate>false</allowCreate>
  <allowDelete>true</allowDelete>
  <allowEdit>false</allowEdit>
  <allowRead>true</allowRead>
  <modifyAllRecords>false</modifyAllRecords>
  <object>Attachment</object>
  <viewAllFields>false</viewAllFields>
  <viewAllRecords>false</viewAllRecords>
</objectPermissions>
```

### 2. Account and Opportunity viewAllRecords
```xml
<objectPermissions>
  <allowCreate>false</allowCreate>
  <allowDelete>false</allowDelete>
  <allowEdit>false</allowEdit>
  <allowRead>true</allowRead>
  <modifyAllRecords>false</modifyAllRecords>
  <object>Account</object>
  <viewAllFields>false</viewAllFields>
  <viewAllRecords>true</viewAllRecords>
</objectPermissions>
<objectPermissions>
  <allowCreate>false</allowCreate>
  <allowDelete>false</allowDelete>
  <allowEdit>false</allowEdit>
  <allowRead>true</allowRead>
  <modifyAllRecords>false</modifyAllRecords>
  <object>Opportunity</object>
  <viewAllFields>false</viewAllFields>
  <viewAllRecords>true</viewAllRecords>
</objectPermissions>
```

### 3. Opportunity.Notes Field Permission
```xml
<fieldPermissions>
  <editable>true</editable>
  <field>Opportunity.Notes</field>
  <readable>true</readable>
</fieldPermissions>
```
