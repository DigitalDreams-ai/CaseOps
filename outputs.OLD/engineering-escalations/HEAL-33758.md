# Engineering Handoff: HEAL-33758

**Issue:** HEAL-33758 — SF- Multiple Lab Order Errors & Correction Request  
**Problem:** Four Lab Order records (LAB-000234000, LAB-000234001, LAB-000233409, LAB-000234287) fail to save with error: "The product(s) that are on the opportunity are not servicable items and do not require a lab order." All are attempting to create Lab Orders for "Genetic Testing - Virtual" product but are blocked by Flow/validation logic.

**Potential Fix:** Investigate Product_Lab_Order_Configuration lookup logic in Lab Order creation flows (Autolaunch_Lab_Order_Get_Serviceable_Item_Details and Autolaunch_Lab_Order_Create_Collection_of_Serviceable_Items_and_Their_Lab_SKU_s). Either: (1) Create missing Product_Lab_Order_Configuration records for affected product/provider combinations, OR (2) Fix incorrect Lab_Provider_Configuration assignments on affected Lab Orders, OR (3) Update Flow logic to handle edge cases (patient accommodations, lab provider overrides).

## Root Cause

**Investigation found:** Validation logic in Lab Order creation flows (specifically the two Autolaunch flows) checks for matching Product_Lab_Order_Configuration__c records keyed by Lab_Provider. 

**Likely causes:**
1. **Missing Product_Lab_Order_Configuration records** for product + lab provider combination on affected Lab Orders
2. **Incorrect Lab_Provider_Configuration assignment** on Lab Order — field may be NULL, set to wrong provider, or not populated before Flow executes
3. **Flow logic error** — Flow may be checking for exact provider match that doesn't account for alternative providers or accommodations
4. **Data inconsistency** — Product_Lab_Order_Configuration records exist in Production for this product with multiple providers, yet Flow cannot locate matching record for affected Lab Orders

Production metadata investigation confirmed:
- Product: "Genetic Testing - Virtual" (01t5a00000GZUJgAAP) IS serviceable (has Product_Lab_Order_Configuration records)
- Product_Lab_Order_Configuration records exist for this product with ASC, Novogenia, Empower, CSS lab providers
- Yet all four Lab Orders fail with "not serviceable" error

**Validation rule formula could not be retrieved via CLI.** Error message suggests validation rule checks Product_Lab_Order_Configuration availability, but exact logic requires code review.
