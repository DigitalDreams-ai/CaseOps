## Engineering Message

**Issue:** HEAL-33583 — SMS-Magic inbound reply notification emailed to wrong scheduling coordinator
**Problem:** When a patient replies via SMS to a text originally sent by scheduling coordinator Jemimah, Salesforce (via SMS-Magic) sends the inbound message notification email to Stephanie Silva (Lead Lab Coordinator) instead of Jemimah. This is a recurring pattern — a prior ticket (HEAL-33537) had the same symptom for Stephanie's own outbound texts, which Idman partially resolved. The fix did not address the systemic cause, and the issue now surfaces for Jemimah.
**Potential Fix:** Audit the SMS-Magic notification flow or workflow rule that determines the recipient of the inbound reply notification email. Identify whether the recipient is resolved from SMS record Owner, a Vonage number-to-user mapping table, or a hardcoded/incorrectly set field. Correct the mapping so each coordinator receives notifications only for SMS threads they originated. Review all scheduling coordinators for similar misconfiguration.

## Root Cause

Not yet confirmed. Hypotheses in priority order:
1. SMS-Magic notification flow resolves the email recipient from the SMS record's OwnerId or a related lookup, and that field is incorrectly populated (e.g., pointing to Stephanie instead of the sending coordinator).
2. Vonage number-to-user mapping table has Jemimah's outbound number (`17866404062`) mapped to Stephanie Silva's user record.
3. Idman's fix for HEAL-33537 corrected Stephanie's record in isolation without addressing the underlying mapping logic, leaving other coordinators vulnerable.
