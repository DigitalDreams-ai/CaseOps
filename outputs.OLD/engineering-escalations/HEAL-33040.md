# Engineering Handoff

## Engineering Message

**Issue:** HEAL-33040 — Patient Account not syncing to Shopify
**Problem:** Accounts created in Salesforce via the new patient link are not being synced to Shopify. The Shopify Customer ID field remains empty and no error appears in the Shopify Sync Message field (silent failure). A manual fallback field (`Trig Shopify Sync`) that should trigger a sync when set to True is also not working. Both the automatic (creation-time) and manual sync paths are broken.
**Potential Fix:** Inspect the Apex trigger(s) and/or Flow(s) on the Account object that handle Shopify sync. Identify why the creation-time sync does not fire (or fails silently) for accounts created via the new patient link entry point, and why the `Trig Shopify Sync` field update does not invoke the sync. Fix the broken logic and ensure both paths write a Shopify Customer ID and a status message to the Account record on success.

**Affected record IDs:** Account `001Ql00000x8VCkIAM` (confirmed affected; other accounts created via new patient link during the same period may also be affected)

## Root Cause

Unknown — requires Engineering investigation. Hypotheses in priority order:
1. The Apex trigger or Flow on Account that calls the Shopify sync API is not firing for accounts created via the new patient link (possible entry-point-specific condition or bypass).
2. The sync logic is firing but encountering an unhandled exception or callout failure that is swallowed silently (no error written to `Shopify Sync Message`).
3. The `Trig Shopify Sync` field handler (likely a separate Flow or trigger branch) is broken independently — suggesting the core sync callable logic itself may be the failure point shared by both paths.
