The problem is not the PDF template. The DEA line is blank on the generated PDF because that field was never populated during order creation. The gap is in the autolaunched flow ‘Opportunity_to_Order'. The Get Opportunity Fields element queries DEA_Number__c, State_License_Number__c, and Patient_Driver_s_License__c, but the Assign Order Fields element never assigns those values onto the new Order. This is a subflow of the screenflow: 'Screenflow - Confirmation to Create Pharmacy Order’ Steps to reproduce: On an Opportunity record, click on the button ‘Generate Pharmacy Order’ Example: https://10xhealth.lightning.force.com/lightning/r/Order/801Ql00000v0QZdIAM/view A Purchase Order with Scripts pdf will be generated with a missing DEA Number.
