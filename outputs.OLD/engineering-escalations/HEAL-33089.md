# Engineering Escalation — HEAL-33089

**Issue:** HEAL-33089 — Precision Nutrition error
**Problem:** Novogenia lab order integration fails to persist order ID back to Lab Order record. Apex code successfully submits order to Novogenia API and receives response with order number, but fails to parse response JSON and/or write order number to Lab_Order__c.Lab_Order_Id__c field.
**Potential Fix:** Audit and fix JSON response parsing in gt_Callouts.submitOrder (line 77) to match current Novogenia API response schema, and implement missing field update logic to persist externalOrderNo/orderNo to Lab_Order_Id__c field.

## Root Cause

**Primary:** Novogenia API response JSON parsing fails at gt_Callouts.submitOrder line 77. The code attempts to deserialize Novogenia's escaped JSON string (response.data.labResponse) into gt_DTO_OrderResponse_Details class, but deserialization throws an exception. Likely cause: Novogenia API response schema has changed or response structure doesn't match expected DTO fields (externalOrderNo, orderNo, customerNo, externalOrderTimestamp, customerDetails, orderedProducts, delivery, questions).

**Secondary (Implementation Gap):** Even if JSON parsing succeeds, no code found that writes the parsed order number (externalOrderNo or orderNo) back to Lab_Order__c.Lab_Order_Id__c field. Codebase search found ZERO references to Lab_Order_Id__c field updates. The field exists and is designed as external ID to store "the ID that comes back from the lab," but persistence logic is missing or incomplete.

**Tertiary:** Exception is caught and logged silently to Nebula Logger; user sees generic error message without actionable detail.

## Potential Fix

1. **Audit Novogenia API Response Schema:** Determine current Novogenia API response format for lab order submissions. Compare with gt_DTO_OrderResponse_Details DTO structure. Identify missing, renamed, or restructured fields (focus on externalOrderNo, orderNo, externalOrderTimestamp, customerDetails).

2. **Fix JSON Parsing Logic:** Update gt_Callouts.submitOrder line 77 deserialization to handle current Novogenia response format. May require:
   - Adding null-checks for optional Novogenia fields
   - Updating DTO class definitions to match current schema
   - Fixing datetime parsing if externalOrderTimestamp format changed
   - Improving escaping/unescaping logic if Novogenia response format changed

3. **Implement Field Persistence:** After successful parsing at line 77, add code to update Lab Order record with order number:
   ```apex
   // After line 78 (successful parsing):
   if (response.data.labDetails != null && response.data.labDetails.externalOrderNo != null) {
       labOrder.Lab_Order_Id__c = response.data.labDetails.externalOrderNo;
       update labOrder;
   }
   ```

4. **Improve Error Messaging:** Log detailed exception info (field names, expected vs. actual JSON structure) to Nebula Logger to aid future debugging.

5. **Test:** Verify fix against sample Novogenia responses (current format). Confirm Lab Order record is updated with order number after successful submission.
