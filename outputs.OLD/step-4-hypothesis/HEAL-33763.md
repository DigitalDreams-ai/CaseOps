# Step 4 — Problem Hypothesis and Solution

**Issue:** HEAL-33763  
**Date:** 2026-05-21

---

## Problem Hypothesis

**Confirmed facts (from Step 3):**
- Tier 1 support staff (Ray Guevarra, Earth Alberio, Genevieve Aguilar, Marianne Sallo) lack three specific permissions
- Currently cannot delete Opportunity attachments, view Andy Haas-owned records, or add notes to closed Opportunities
- Each gap forces escalation to Tier 2 for administrative tasks that add no diagnostic value
- This creates unnecessary queue volume and turnaround delays

**Symptoms (what the user reported):**
- GT Opportunity workflows blocked when file regeneration requires deleting old attachments
- MyWellness API account investigations require Tier 2 reassignment (unnecessary overhead)
- Closed Opportunity documentation incomplete when notes cannot be added without reopening

**Root cause hypothesis:**
Tier 1 Permission Set lacks three object/field-level permissions: Attachment delete on Opportunities, record visibility for Andy Haas-owned Accounts/Opportunities, and note creation on closed Opportunities.

---

## Smallest Viable Fix

**What to fix:**
- **Metadata artifact:** Tier 1 Support Permission Set (existing; name to be confirmed in Production)
- **Change scope:** Add three specific permission grants:
  1. Attachment object: Add Delete permission
  2. Account/Opportunity objects: Add Read permission for Andy Haas-owned records (via sharing rule or "View All" equivalent)
  3. Note field on Opportunity: Add Create/Read permission scoped to closed records only
- **Why it solves the problem:** Direct permission gap removal; each grant unblocks one workflow, eliminating escalation requirement

## Risks and Constraints

**Implementation risks:**
- Granting Attachment delete could allow removal of important files if user judgment is poor (mitigated by Recycle Bin recovery)
- Granting Andy Haas record visibility may unintentionally expose other records if sharing rules are too broad
- Note creation on closed records could create audit trail expectations that conflict with financial record-locking policies

**Constraints:**
- Cannot modify user roles or profile-level permissions; must use Permission Set
- Cannot grant edit access to closed Opportunities (financial data integrity requirement)
- Sharing rules for Andy Haas visibility must match existing org structure

**Mitigation:**
- Recycle Bin ensures file recovery if accidentally deleted
- Test record visibility scope carefully; limit to Accounts/Opportunities object level
- Document note creation policy for Tier 1 (notes only, no record edits)
