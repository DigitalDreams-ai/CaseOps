# Step 4 — Problem Hypothesis and Solution

**Issue:** HEAL-30437  
**Date:** 2026-05-28

---

## Problem Hypothesis

**Confirmed facts (from Step 3):**
- DTC (Main) Shopify storefront orders auto-populate in Salesforce (working)
- Shopify Network orders do NOT auto-populate (non-working)
- Michael Moffo manually releases Shopify Network orders one-by-one (manual workaround)
- Robert Connors noted "affiliate branch was turned off somewhere" (explicit evidence of disabled branch)
- Signifyd fraud validation is automated; integration works; issue is isolation to auto-population step
- Configuration must match between DTC (Main) and Shopify Network

**Symptoms:**
- Shopify Network orders created but not automatically released
- User must manually trigger release process for each order
- Blocks Michael Moffo's workflows; blocks Supply Chain automation

**Root cause hypothesis:**
Order auto-population flow has a condition or branch that excludes Shopify Network order source, or the Shopify Network execution path is explicitly disabled in the flow logic.

---

## Smallest Viable Fix

**Metadata/code artifact:**
- Primary: Flow controlling order auto-population (name TBD; likely contains "order" and "populate" or "release")
- Secondary: Order supplement generation flow (referenced as "generate supp order"; may have related branch)

**Change scope:**
- Enable or create Shopify Network execution branch in the order auto-population flow
- Match DTC (Main) conditional logic for the Shopify Network source
- Ensure supplement order generation also covers Shopify Network

**Why it solves the problem:**
Flow condition currently filters out Shopify Network orders. Enabling the branch aligns Shopify Network with DTC (Main) configuration, restoring auto-population.
