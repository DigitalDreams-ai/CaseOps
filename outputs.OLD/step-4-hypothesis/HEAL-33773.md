# Step 4: Problem Hypothesis and Solution

## Issue
Shopify POS transactions synced to Salesforce Opportunities incorrectly transition to "Waiting for Approval" status instead of auto-approving when payment is collected.

## Root Cause Hypothesis
The Shopify sync automation (likely a Flow or Process Builder) is unconditionally setting Opportunity Status to "Waiting for Approval" without checking the payment_collected flag or transaction state from the Shopify payload. Additionally, the sync may lack idempotency, causing re-syncs to override APPROVED status back to "Waiting for Approval."

## Smallest Viable Fix
1. **Identify the artifact**: Locate the Shopify sync Flow, Process Builder, or Apex automation controlling Opportunity Status updates
2. **Add conditional logic**: Modify the automation to:
   - Check if payment was collected in Shopify payload (payment_collected = true or equivalent)
   - Only set Status to "APPROVED" if payment collected
   - Skip status update if Opportunity already in APPROVED (idempotency check)
3. **Respect prior state**: Prevent re-sync from overriding APPROVED → "Waiting for Approval"

## Sandbox Validation Plan
1. Sync a test Opportunity record with payment_collected = true from Shopify
2. Verify Status auto-transitions to APPROVED
3. Re-sync the same record
4. Verify Status remains APPROVED (not reverted to "Waiting for Approval")
5. Test with payment_collected = false
6. Verify Status stays in appropriate state (not forced to "Waiting for Approval")

## Rollback Plan
- Revert the sync automation to prior version via version control or Salesforce Flow/Process activation toggle
- Manual status corrections for affected Opportunities if needed

## Risks and Constraints
- May impact other syncs from Shopify if the automation is shared
- Approval process requirements may need review (ensure conditional approval logic is preserved)
- Data integrity check needed on existing APPROVED Opportunities to ensure they're not re-corrupted

## Production Deploy Readiness
- Gearset deployment required (Flow/Process metadata changes)
- Change control procedure: Test in Sandbox, then promote via Gearset to Production
- Monitoring: Check sync logs post-deployment for transaction status patterns
