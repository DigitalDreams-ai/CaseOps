**Root Cause:** Salesforce Metadata API limitation
- MDAPI cannot update existing PermissionSet metadata (only create new ones)
- PermissionSet Tier_1 (ID: 0PSRh00000025ObOAI, Label: "Tier 1") already exists in Sandbox
- Attempting to deploy updated XML triggers "label in use" error because the system cannot update an existing permission set via MDAPI

**This is a Known Salesforce Limitation:** PermissionSet objects lack update support in the Metadata API. See internal notes for full context.

- The system interprets a redeploy of the same permission set as a label conflict (duplicate "Tier 1" label)
- This is a documented Salesforce API constraint, not a data or configuration error
