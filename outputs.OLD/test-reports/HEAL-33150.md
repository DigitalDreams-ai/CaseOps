# Test Report - HEAL-33150

## Status

No fix has been implemented or validated. All functional tests remain blocked pending Engineering/Admin changes.

## Current Validation

Production read-only validation was performed:

- The affected Cx Case has outbound emails logged, but no inbound internal approval reply.
- Other Customer Experience cases do show incoming replies when those replies reach routed mailboxes such as `support@10xhealthsystem.com` or `info@10xhealthsystem.com`.
- This supports a reply-path/rooting issue, not a global Cx threading failure.
- The exact affected action is `Case.SendEmail` on layout `Case-Customer Experience`.
- The action exposes editable `ValidatedFromAddress` and does not enforce a Salesforce-routed From/Reply-To.
- The affected outbound used `ValidatedFromAddress = ntorres@10xhealthsystem.com`, with `EmailTemplateId = null` and `EmailRoutingAddressId = null`.

## Required Sandbox Tests

### Test 1: Reproduce Current Internal Approval Failure

1. Create or use a Sandbox Customer Experience Case.
2. Send the same internal approval email workflow used in Production.
3. Inspect the Outlook message's From and Reply-To.
4. Reply from Outlook.
5. Verify whether Salesforce creates an `Incoming = true` EmailMessage on the original Case.

Expected before fix: reply does not attach if it is not delivered to Salesforce.

### Test 2: Validate Corrected Reply Path

1. Apply the selected routing fix in Sandbox.
2. Send the same internal approval email again.
3. Confirm Outlook receives a Salesforce-routed Reply-To or equivalent forwarded mailbox path.
4. Reply from Outlook.
5. Verify the reply attaches to the original Case as `Incoming = true`.

Expected after fix: reply attaches to the original Case.

### Test 3: No Duplicate Or Orphan Case

1. After Test 2, search for Cases created from the test reply sender/subject.
2. Confirm no duplicate or orphan Case was created.

Expected: only the original Case contains the inbound reply.

### Test 4: Existing Routed Mailbox Regression

1. Send a controlled email/reply flow through an existing routed mailbox such as `support@10xhealthsystem.com` or `info@10xhealthsystem.com`.
2. Confirm inbound replies still attach to the correct Case.

Expected: existing routed flows remain unchanged.
