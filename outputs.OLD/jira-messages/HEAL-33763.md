## [INTERNAL]

Root cause confirmed: Tier_1 PermissionSet (ID: 0PSRh00000025ObOAI, label: "Tier 1") in Production is missing three permission configurations:

1. Attachment object: allowDelete=false (needs true)
2. Account object: viewAllRecords=false (needs true)
3. Opportunity object: viewAllRecords=false (needs true)
4. Opportunity.Notes field: editable=false or missing (needs readable=true, editable=true)

Identified via SOQL PermissionSet query (confirmed PS exists) and inference from 30+ similar Tier support permission sets in HEAL-33562 metadata dump.

**Fix:** Three permission elements added to source XML (force-app/main/default/permissionsets/Tier_1.permissionset-meta.xml). MDAPI deployment fails due to API limitation (label conflict on PS update), so manual application required via Setup UI or Gearset.

**Deployment path:** Admin applies manually to 10xhealth-sean Sandbox, runs three validation tests (file delete, Andy Haas record view, note creation on closed Opp), then deploys to Production via Gearset.

**Status:** Awaiting admin manual Sandbox application. No Engineering code escalation needed. Configuration-only fix.
