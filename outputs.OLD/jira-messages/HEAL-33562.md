# Jira Message Draft - HEAL-33562

## Issue Summary
Tyler Kaplan reported that the "Recall" option for approved Opportunities in the Approval History is missing, preventing her from recalling approvals that need editing.

## Root Cause Identified
Investigation revealed that two Salesforce flows automatically submit Opportunities for approval without properly specifying the submitter context. When approvals are submitted by automation without designating the record owner as the submitter, Salesforce only allows the automated system user (not the record owner) to recall. This prevents Tyler and other Nurses from seeing and using the Recall button.

## Fix Deployed
We've updated both automation flows to designate the record owner as the submitter when submitting for approval. This restores the ability for Nurses and other users to recall their own approvals through the standard Approval History interface.

**Flows Updated:**
- Opportunity Shopify Automations (affects all Shopify-originated opportunities)
- Opportunity Superhuman Protocol Automations (affects Superhuman Protocol opportunities)

## Next Steps
The fix has been deployed to our Sandbox environment for testing. Once validated, it will be deployed to Production. We expect this to fully resolve the issue and allow you to recall approvals as before.

**Timeline:** Testing in progress; Production deployment expected this week.
