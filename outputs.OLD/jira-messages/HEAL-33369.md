Hi Mya,

I investigated your Time to Schedule report issue and have resolved it in our Sandbox environment.

**Root cause:**

The Opportunity Field History report could not display Account-level information like billing state and main practitioner because those fields only exist on the Account side. Salesforce's report structure doesn't automatically expose parent Account fields in Opportunity reports. The workaround—manually cross-referencing data in Excel—introduced error risk and extra work.

**What was changed:**

I created two new data fields on the Opportunity object that pull the billing state and practitioner information directly from the parent Account. These fields now appear in the report, allowing you to view Time to Schedule alongside the state and provider segments you need, all in one place.

**Where this fix lives:**

The fix was **validated in Sandbox only**. **Production has not been updated** by this work. The updated reports are ready to be moved to Production through our standard deployment process. Once that happens, all three reports (Lab Review, Genetic Test Clarity Calls, and Peptide Express Consultations) will include the billing state and practitioner columns.
