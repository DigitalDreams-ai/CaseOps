   - **Hypothesis:** Validation rule may be checking if a matching Product_Lab_Order_Configuration__c record exists. If none found, it treats product as "not serviceable"

2. **Lab Provider Dependency:**
   - Both flows check `Lab_Provider_Configuration__r.Id` (lookup from Lab_Order to Lab Provider Configuration)
   - [Field: Lab_Order__c.Lab_Provider_Configuration__r](sf://field/Lab_Order__c/Lab_Provider_Configuration__r) is required in layout
   - Flows retrieve Product_Lab_Order_Configuration__c WHERE Lab_Provider__c = Lab_Provider_Configuration.Id
   - **Implication:** If Lab_Provider_Configuration is NULL or set to wrong provider, validation rule may not find matching Product_Lab_Order_Configuration, triggering "not serviceable" error

   - LAB-000233409 explicitly requests "Novogenia lab instead of ASC" — suggests Lab_Provider_Configuration was set incorrectly, and no matching Product_Lab_Order_Configuration exists for Novogenia provider
   - LAB-000234287 (patient needing swab kit) also likely lacks Product_Lab_Order_Configuration for required accommodations
