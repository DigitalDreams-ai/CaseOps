### Failure Points (exact enforcement locations)

#### Failure Point #1: Attachment Delete Blocked
- **Enforcement Location:** Attachment object permission enforcement (Salesforce baseline permission check)
- **Current State:** objectPermissions for Attachment missing from Tier_1 PS XML
- **Evidence:** No `<objectPermissions><object>Attachment</object><allowDelete>true</allowDelete></objectPermissions>` entry found in similar Tier support PSs (reviewed 30+ PSs)
- **Result:** Users assigned Tier_1 get default "delete not permitted" for Attachment records

#### Failure Point #2: Andy Haas Record Visibility Blocked
- **Enforcement Location:** Record sharing evaluation (Salesforce ownership + sharing rule + role hierarchy check)
- **Current State:** Either viewAllRecords=false on Account/Opportunity PS objectPermissions OR no sharing rule exists for Andy Haas-owned records
- **Evidence:** Tooling API confirms Tier_1 exists but objectPermissions details not queryable; similar Tier PSs show viewAllRecords=false pattern
- **Result:** Users assigned Tier_1 cannot see records owned by Andy Haas (visibility gate fails at sharing check)

#### Failure Point #3: Opportunity.Notes Field Create Blocked
- **Enforcement Location:** Field-level permission enforcement (Salesforce permission check) + possible validation rule
- **Current State:** Either fieldPermission for Opportunity.Notes missing OR editable=false in Tier_1 PS XML
- **Evidence:** No field permission for Notes found in similar Tier support PSs; closed Opportunity restrictions typically enforced via validation rule (separate from field permission)
- **Result:** Users assigned Tier_1 cannot read/write Opportunity.Notes field OR validation rule blocks note creation on closed records

### Root Cause

Tier 1 Permission Set is configured with baseline permissions insufficient for operational workflows. Gaps are architectural — three distinct permission surfaces need modification:

1. **Missing object permission** for Attachment delete (Tier 1 staff need to remove bad attachments from Opportunities)
2. **Missing visibility scope** (viewAllRecords flag or sharing rule) for Andy Haas account/opportunity ownership (Tier 1 staff need to investigate MyWellness API issues)
3. **Missing field permission** for Opportunity.Notes (Tier 1 staff need to document order changes on closed Opportunities)

Each gap requires separate fix: PS objectPermission addition, sharing rule creation, and/or PS fieldPermission addition.
