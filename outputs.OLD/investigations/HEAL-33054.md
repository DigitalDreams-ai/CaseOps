## Issue Understanding

### Observed Behavior
Ashlee Edwards and team need to track which patient Account records represent Cardone Ventures employees. Currently, there is no way to mark or identify these accounts, preventing reporting across opportunity records.

### Expected Behavior
- A checkbox field called "Cardone Ventures Employee" exists on the Account object (patient account record type)
- When the checkbox is selected on an Account, the flag automatically propagates to all related Opportunity records
- All stakeholders (Medical Coordinators, Nursing Staff, Operations) can view and report on this flag
- Data can be pulled for reporting purposes across all opportunity record types

### Acceptance Criteria
1. Checkbox field "Cardone_Ventures_Employee__c" created on Account object, default = false
2. Formula field "Cardone_Ventures_Employee__c" created on Opportunity object with formula: Account.Cardone_Ventures_Employee__c
3. Field added to Account page layouts (Account Summary section, after Do_Not_Sync_to_Biocanic__c)
4. Field added to all 5 PersonAccount page layouts (after Brand__c or VIP_Account__c)
5. Field added to all 31 in-use Opportunity page layouts (after AccountId in Opportunity Information section) as read-only
6. Appropriate FLS permissions set per permission set: Account (read-only for most, read+edit for Admin_Team and Medical_Coordinator); Opportunity (read-only for all except Admin_Team)
7. Cardone Ventures employee Accounts identified manually and checkbox marked; Opportunity records auto-populate via formula
