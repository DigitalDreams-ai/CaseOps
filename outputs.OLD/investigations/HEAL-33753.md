### Failure Point (where in the flow it breaks)

The dispatcher flow [Flow: Record_Trigger_After_Save_Send_Genetic_Breakthrough_Schedule_Link](sf://3005a000000H3YWAA0) evaluates its entry filter `NOT(ISNULL({!$Record.PricebookEntry.Product2.Calendly_Email_Template_Id__c}))` on each new OpportunityLineItem. For opportunities with "At-Home Blood Test: Hormone Panel Consultation" product, this evaluates **false** because `Calendly_Email_Template_Id__c` is NULL — so the flow never runs and no email is sent. Independently, even if the flow did fire and reference template `00XQl000005kQavMAE`, the recipient would receive an email containing the literal text "INSERT MERGE FIELD FOR CALENDLY BOOKING" instead of a clickable scheduling link.

**Two-stage failure:**
1. **Flow entry gate fails:** Product2.Calendly_Email_Template_Id__c is NULL on "At-Home Blood Test: Hormone Panel Consultation". Result: flow skipped, zero emails sent.
2. **Template content gap:** Email template body contains placeholder string instead of the actual Calendly URL (`https://calendly.com/d/cw6m-rd6-8rd` with prefill params).

### Root Cause

Configuration incomplete on two fronts:
1. **Email template skeleton** (`00XQl000005kQavMAE`): Ashlee Edwards created the template but left the Calendly URL as placeholder text "INSERT MERGE FIELD FOR CALENDLY BOOKING" — awaiting exact URL to insert.
2. **Product2 link missing**: The existing "At-Home Blood Test: Hormone Panel Consultation" product has the Calendly booking link (`https://calendly.com/d/cw6m-rd6-8rd`) but is not wired to the email template (`Calendly_Email_Template_Id__c` is NULL).

Result: Dispatcher flow never fires because template ID is missing → no emails sent. The supporting infrastructure (flow, fields, OrgWideEmailAddress) is fully in place and unchanged — this is purely a missing config + missing data linkage, not a metadata or code defect.
