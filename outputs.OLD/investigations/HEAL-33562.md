## Salesforce Problem

### Confirmed Root Cause

**PRIMARY ISSUE:** Two flows auto-submit Opportunities for approval WITHOUT specifying the `submittedBy` parameter:

1. `Record_Trigger_Opportunity_Shopify_Automations` (ALL Shopify opportunities)
   - Location: force-app/main/default/flows/Record_Trigger_Opportunity_Shopify_Automations.flow-meta.xml
   - Problem: Submit action missing submittedBy parameter
   - Impact: System user becomes submitter, blocks owner recall

2. `Record_Trigger_Opportunity_Superhuman_Protocol_Submit_For_Approval` (Superhuman Protocol Opportunities)
   - Location: force-app/main/default/flows/Record_Trigger_Opportunity_Superhuman_Protocol_Submit_For_Approval.flow-meta.xml
   - Problem: Submit action missing submittedBy parameter
   - Impact: System user becomes submitter, blocks owner recall

**Why it fails:** Salesforce approval processes only allow the original SUBMITTER to recall an approval. When flows submit without specifying `submittedBy`, Salesforce defaults submitter to system/automation user. Owner cannot recall because they are not the submitter.

**Why Order Processing works:** Order Processing users likely submit directly via UI (not automation), making themselves the submitter and able to recall.

### Approval Process Configuration (Verified)
- **allowRecall:** true (globally enabled)
- **allowedSubmitters:** owner, Back_Office role, Operations_Manager_new role
- **RecallActions:** Update_Stage_Open (returns to Open)

## Solution Plan

### Classification
Support-resolvable (flow fix, no Engineering escalation)

### Proposed Change
Add `submittedBy=$Record.OwnerId` parameter to submit actions in both flows:

```xml
<inputParameters>
    <name>submittedBy</name>
    <value>
        <elementReference>$Record.OwnerId</elementReference>
    </value>
</inputParameters>
```

Files to modify:
1. Record_Trigger_Opportunity_Shopify_Automations.flow-meta.xml (Submit_for_Approval action)
2. Record_Trigger_Opportunity_Superhuman_Protocol_Submit_For_Approval.flow-meta.xml (Send_to_Approval action)

### Why This Fixes It
Setting `submittedBy=$Record.OwnerId` makes the record owner the submitter in Salesforce's eyes. Owner can then recall their own submissions.

### Risks
- None—this is standard Salesforce best practice for automation-submitted approvals
- Maintains record owner as submitter context
- No permission/role changes needed

### Rollback Plan
Revert flow changes if unexpected behavior observed.
