## Confirmed Facts

- Production org used for metadata/read-only checks: `10xhealth`.
- The screen flow `Order_Screenflow_Submit_Pharmacy_Order` does not map address fields. It calls Apex action `PharmacyOrderValidation`, then Apex action `PharmacySubmitPrescriptionAction`, passing only `recordId`.
- `PharmacySubmitPrescriptionAction` uses callout config `Wellvi_Submit_Prescription` and instantiates `PharmacySubmitPrescriptionCallout`.
- `PharmacySubmitPrescriptionCallout` extends `ApiOperation`; the request body is dynamically built from `API_Operation__mdt`, `API_Parameter__mdt`, and `API_Child_Parameter__mdt`.
- `PharmacyOrderValidation` validates these custom Order fields: `Shipping_Street__c`, `Shipping_City__c`, `Shipping_State__c`, `Shipping_Zip_Code__c`.
- Read-only Production queries confirmed Order `801Ql00000zolrtIAA` has all four custom shipping fields populated. A separate check also confirmed the standard shipping and billing address fields are populated. Raw address values were not stored in repo output.
- The live Wellvi CMT maps:
  - `Shipping_City__c` -> `$.shippingAddress.city`
  - `Shipping_Street__c` -> `$.shippingAddress.addressLine1`
  - `Shipping_State__c` -> `$.shippingAddress.stateProvince`
  - `Shipping_Zip_Code__c` -> `$.shippingAddress.postalCode`
- `JsonUtility.putByPath` correctly creates nested map paths from JSON paths such as `$.shippingAddress.city`.

## Root Cause Assessment

The originally suspected "SOQL missing address fields" path is not supported by the retrieved metadata. The generic `ApiOperation` dynamic query will include the four custom shipping fields because the `API_Parameter__mdt` rows reference them directly.

The likely defect is a payload contract mismatch between the Salesforce CMT JSON paths and the TenX/Wellvi endpoint model. Salesforce is sending address fields under lowercase nested paths such as `shippingAddress.city`, while the API error names the required model fields as `City`, `PostalCode`, `AddressLine1`, and `StateProvince`. Engineering needs to verify the exact API contract for `/pharmacies/wellvi/prescriptions` and update either the CMT JSON paths or the downstream DTO/binding so these fields bind correctly.

## Solution Implemented

**Classification:** Support-resolvable (metadata configuration update)

**Root Cause Confirmed:** The Wellvi/TenX API contract expects address fields as root-level PascalCase properties (`City`, `PostalCode`, `AddressLine1`, `StateProvince`), but the CMT JSON paths were mapping them to nested lowercase paths under `shippingAddress`.

**Fix Applied (Support path):**

Updated four `API_Parameter__mdt` records in Sandbox to map address fields to correct JSON paths:

| Salesforce Field | Old JSON Path | New JSON Path | Reason |
|---|---|---|---|
| `Shipping_City__c` | `$.shippingAddress.city` | `$.City` | API contract requires PascalCase at root |
| `Shipping_Street__c` | `$.shippingAddress.addressLine1` | `$.AddressLine1` | API contract requires PascalCase at root |
| `Shipping_State__c` | `$.shippingAddress.stateProvince` | `$.StateProvince` | API contract requires PascalCase at root |
| `Shipping_Zip_Code__c` | `$.shippingAddress.postalCode` | `$.PostalCode` | API contract requires PascalCase at root |
