## Issue Understanding

### Observed Behavior

Currently, there is no documented capability to report on escalation timestamps and elapsed time metrics in the Case reporting system. The customer support team cannot track:
- The date/time when cases transition from Tier 1 (call center) to Tier 2 (internal agents)
- Elapsed time from case open to escalation
- Elapsed time from escalation to case closure

### Expected Behavior

Salesforce Case reporting should provide visibility into escalation timelines:
- Capture and store the exact date/time a case is escalated from Tier 1 to Tier 2
- Enable reports showing elapsed time from case creation to escalation
- Enable reports showing elapsed time from escalation to case resolution
- Support Customer Support department reporting needs for SLA tracking and performance analysis

### Acceptance Criteria

1. Cases record the escalation date/time (from Tier 1 to Tier 2 handoff)
2. Reports can calculate elapsed time from case open to escalation
3. Reports can calculate elapsed time from escalation to case closed
4. Multiple teams and departments can access escalation reporting
5. Solution aligns with existing Case field configurations and FLS permissions

## Problem Location (Step 6 — Exact Artifacts)

### Artifact 1: Flow — Record_Trigger_Case_Assign_Round_Robin_Cases

**Location:** Production, API Name: `Record_Trigger_Case_Assign_Round_Robin_Cases`
**Type:** Record-Triggered Flow (AutoLaunchedFlow)
**Trigger:** `RecordAfterSave` on Case object, `CreateAndUpdate`
**Activation Status:** Active (Status = Active, Trigger Order 400)
**Filter Formula:** Applies only to Customer Experience record type AND status NOT Closed/Completed/Merged

**Escalation Trigger Logic (lines 589-671 in flow XML):**
- Decision "Should_Flow_Trigger" has 3 rules:
  1. **"Is this an automated process"** — fires for new non-automated Cases → routes to Tier 1 (Primary Owner assignment)
  2. **"Escalated Case Tier Two"** — fires when `Dispo_Level_3__c = "Escalation"` AND `EscalatedTrigger = true` AND `IsRecordNew = true` → routes to Tier 2
  3. **"New Case Tier One"** — fires for new Cases where `Dispo_Level_3__c != "Escalation"` → routes to Tier 1

**Where Escalation Timestamp SHOULD Be Populated:**
- **Decision Point:** When `Dispo_Level_3__c` transitions to "Escalation" (Rule: "Escalated Case Tier Two")
- **Current Flow Action:** Route to `Log_Debug_Tier_Two_Path_Executed` → `Get_Tier_Two_Agents` → loop/assign to escalated case owner (`Escalated_Case_Owner__c`)
- **MISSING ACTION:** No flow element populates a timestamp field when escalation is triggered. The flow updates `OwnerId` and `Escalated_Case_Owner__c` but does NOT populate any `Escalation_DateTime__c` field
- **Implementation Surface:** Add a new Record Update action after `Update_Records_1` (which assigns escalated case owner) to populate `Escalation_DateTime__c = {!$Flow.CurrentDateTime}` when entering Tier 2 path

**Failure Point:** Line 914-925 in flow XML — the `Update_Records_1` action (Tier 2 assignment) only updates `Escalated_Case_Owner__c` and `OwnerId`. No timestamp is captured at this critical escalation moment.
