### OrderItem Object Structure
- **Object:** OrderItem (standard Salesforce object)
- **Tracking Field:** [Field: OrderItem.Tracking_Number__c](sf://field/OrderItem/Tracking_Number__c) exists and is permissionable
- **Field Type:** Custom field (permissionable: true, nillable: true)
- **Page Layout:** [Page Layout: OrderItem-Order Product Layout](sf://page-layout/OrderItem/Order%20Product%20Layout)
  - Contains Tracking_Number__c in "Shipping Information" section (line 124 of layout metadata)
  - Layout shows field with Edit behavior for users with proper FLS permissions

### Wellness Team Permission Set Configuration
- **Permission Set Name:** [Permission Set: Wellness_Team](sf://permission-set/Wellness_Team)
- **API Name:** Wellness_Team
- **OrderItem Field-Level Security (FLS) Permissions:**
  - **Found:** OrderItem.Notes__c (Read access only)
  - **Missing:** OrderItem.Tracking_Number__c (NO FLS entry = field is hidden regardless of layout)
  - Total FLS records for OrderItem in Wellness_Team: 1 (only Notes__c)

### Why Wellness Concierge Users Cannot See Tracking Section
Field-Level Security (FLS) acts as a gatekeeper. Even though:
1. The Tracking_Number__c field exists on the OrderItem object
2. The field is placed on the default page layout in the "Shipping Information" section
3. The field has Edit behavior defined on the layout

**Wellness_Team permission set does NOT have FLS access to OrderItem.Tracking_Number__c**, so:
- Salesforce hides the field from the page layout for users with Wellness_Team permission set
- The entire Shipping Information section is effectively empty for these users if Tracking_Number__c is the only field in that section

## Problem Location (Step 6)

### Problem Type

**Access Control — Missing Field-Level Security Permission**

### Specific Artifact

- **Name:** [Field: OrderItem.Tracking_Number__c](sf://field/OrderItem/Tracking_Number__c)
- **API Name:** Tracking_Number__c
- **Type:** Field (custom field on standard object)
- **Parent Object:** OrderItem

### Location in Production

Setup > Objects and Fields > Object Manager > OrderItem > Fields & Relationships > Tracking_Number__c
- Or via Permission Set: [Permission Set: Wellness_Team](sf://permission-set/Wellness_Team) > FLS Records for OrderItem

### Failure Point

**Permission Enforcement — Field-Level Security gate**

The Tracking_Number__c field is hidden from users with [Permission Set: Wellness_Team](sf://permission-set/Wellness_Team) because the permission set has no FLS entry granting read or edit access to this field. Salesforce enforces FLS before rendering layout fields, so the field never appears on the page layout for these users.

### Root Cause

[Permission Set: Wellness_Team](sf://permission-set/Wellness_Team) was created or last updated without including FLS permission for [Field: OrderItem.Tracking_Number__c](sf://field/OrderItem/Tracking_Number__c). The permission set grants access to OrderItem.Notes__c only, leaving the Tracking_Number__c field inaccessible.

**Impact:** Wellness Concierge team members assigned the Wellness_Team permission set cannot view the Wellvi order tracking information displayed in the OrderItem record's Shipping Information section.
