## Issue Understanding

### Observed Behavior

1. Shopify POS transactions synced to Salesforce Opportunities are transitioning to "Waiting for Approval" status when they should auto-approve.
2. Some transactions move from OPEN → WAITING FOR APPROVAL (should be APPROVED since payment collected).
3. Some transactions regress from APPROVED → WAITING FOR APPROVAL (should remain APPROVED).
4. Impact: Clinical department blocked; affects multiple teams and large number of records.

1. When a Shopify POS transaction is synced with payment collected, the Opportunity Status should automatically transition to "APPROVED".
2. Once an Opportunity is in "APPROVED" status, it should persist in that status and not revert to "Waiting for Approval".
3. Synced transactions should reflect their actual payment/approval state from Shopify, not re-trigger approval workflows.

### Flow Analysis: [Record_Trigger_Opportunity_Shopify_Automations](sf://300Rh000002XrSdIAK)

**Key Finding:** Flow unconditionally submits Opportunities for approval via Salesforce approval process, which automatically sets Status to "Waiting for Approval" on submission. The Flow does NOT check payment_collected flag before submission.

**Critical Code Paths:**

1. **Approval Submission (lines 603-631):** Flow contains "Submit_for_Approval" action that calls the built-in `submit` action type with no conditional logic based on payment status or Shopify payload fields
   - Submits `$Record.Id` to approval process
   - No check for `payment_collected` field
   - Executes unconditionally for matching transaction sources

2. **Decision Logic (From_Which_System_Continue_Criteria, lines 917-1041):** Routes Opportunities based on:
   - `Transaction_Source__c` (POS, Store, Direct Sale)
   - Product family counts (Blood Testing, IV Therapy, Genetic Testing, Memberships)
   - Does NOT evaluate payment status from Shopify payload
   - Routes Blood Testing POS/DTC transactions to approval submission (lines 970, 1022)
   - Routes IV POS to "Closed Won" directly (line 992)
   - Routes Memberships to "Approved" directly (line 1037)

3. **Idempotency Issue (lines 1069-1095):** Decision "Is_Triggering_Opp_still_in_Approved_Stage" runs on 2-minute delay but only checks record type, not approval process state. No logic prevents re-submission if re-synced.

### Approval Process Behavior

**Opportunity_Approval_Process (lines 88, 169-182):**
- Initial submission action "Update_Stage_Waiting_For_Aproval" (line 179) unconditionally sets Opportunity Status to "Waiting for Approval"
- Final approval action "Up_Stage_Approved" (line 153) sets Stage to Approved
- No entry criteria check for payment status; only checks RecordType and Shipping_Country

**Genetic_Approval_Process (lines 76, 129-142):**
- Identical behavior: submits to "Waiting for Approval" status
- Entry criteria: Genetic Testing RecordType + shipping address validation; no payment flag check

### Root Cause Confirmed

The Flow submits qualifying Shopify transactions to approval without evaluating whether payment was collected. The approval process mechanism itself automatically sets Status to "Waiting for Approval" upon submission. Additionally, the Flow lacks idempotency checks — re-syncs of the same Opportunity can trigger re-submission, overriding any manually approved status.

## Problem Location (Step 6 — filled after Production metadata drilling)

### Problem Type

**Integration/Process Automation Issue** — Conditional Logic Gap in Shopify Sync Flow

### Specific Artifact

- **Name:** [Record_Trigger_Opportunity_Shopify_Automations](sf://300Rh000002XrSdIAK)
- **API Name (if applicable):** Record_Trigger_Opportunity_Shopify_Automations
- **Type:** Flow (Auto-Launched, Record-Triggered)
- **Status:** Active
- **Production FlowDefinition Id:** `300Rh000002XrSdIAK`
- **Production active Flow version Id:** `301Ql00000w7gqlIAA`
- **Sandbox active Flow version Id after fix:** `301Ea00001Tu99AIAR`
- **Trigger:** RecordAfterSave on Opportunity (Create only)
- **Filter:** `(Transaction_Source__c = 'POS' OR 'Store') AND Transaction_Type__c = 'Shopify'`

### Location in Production

**Setup > Flows > [Record_Trigger_Opportunity_Shopify_Automations](sf://300Rh000002XrSdIAK)** (or via Metadata API as Flow:Record_Trigger_Opportunity_Shopify_Automations)

Flow XML path: `flows/Record_Trigger_Opportunity_Shopify_Automations.flow-meta.xml`

### Failure Point (where in the flow it breaks)

**Decision: "From_Which_System_Continue_Criteria" (lines 917-1041)** routes transactions to "Submit_for_Approval" action without evaluating payment status:

- Blood Testing POS/DTC rule (lines 945-972): Routes to demographic data matching loop, then to approval submission
- Genetic Testing POS rule (lines 997-1024): Routes directly to approval submission
- The flow never checks for a `payment_collected` field or equivalent in the Shopify payload

**Action: "Submit_for_Approval" (lines 603-631)** executes unconditionally, triggering the Opportunity approval process, which automatically sets Status to "Waiting for Approval".

**Missing Idempotency:** Flow runs on RecordAfterSave trigger with 2 delayed scheduled paths (1 min and 2 min delays) but does not prevent re-submission if the Opportunity is updated again with new Shopify data before approval is completed or manually overridden.

### Root Cause

1. **No Payment Status Validation:** Flow Decision logic does not include a condition checking whether `payment_collected = true` (or equivalent Shopify field) before routing to approval submission. Transactions are submitted for approval regardless of payment state.

2. **Approval Process Auto-Sets Status:** When Flow submits Opportunity to approval, the approval process mechanism automatically updates StageName to "Waiting for Approval" via field update action. This is Salesforce approval process behavior — submission triggers the status change immediately.

3. **Lack of Idempotency Protection:** Flow does not check if Opportunity is already in an approved state before re-submitting. Re-syncs from Shopify (or manual record updates matching the trigger filter) can cause previously APPROVED Opportunities to revert to "Waiting for Approval".

**Example Sequence:**
1. Shopify POS transaction synced → Opportunity created, submitted for approval → Status = "Waiting for Approval"
2. Approver manually reviews and approves → Status = "APPROVED"
3. Re-sync from Shopify or record update triggers Flow again → Flow submits to approval again → Status reverts to "Waiting for Approval"

### Opportunity Payment-Related Fields Found

**Field Inventory:**

| Amount (Standard) | Amount | Currency | Standard Opportunity amount field | Populated (473.99, 449.10) |

### Critical Finding: Missing Amount_Paid Data

**Test Records Query Results:**
- Record 1 (006Ql00000eW69hIAC): StageName="Lab Received", Amount=473.99, **Amount_Paid__c=NULL**, Payment_Type="Full"
- Record 2 (006Ql00000f19G9IAI): StageName="Approved", Amount=449.10, **Amount_Paid__c=NULL**, Payment_Type="Full"

**Root Issue:** The Flow references only `$Record.Transaction_Source__c`, `$Record.Transaction_Type__c`, `$Record.StageName`, and `$Record.RecordType` — **it does NOT check `Amount_Paid__c` or any payment field**. The Flow unconditionally routes Shopify POS/DTC transactions to approval submission regardless of whether payment was collected.

**Payment Status Field Recommendation:**

To fix the conditional logic, the Flow must be updated to check:
- **`Amount_Paid__c > 0`** OR
- **`Amount_Paid__c >= Amount`** (payment amount equals or exceeds order amount)

This condition should gate the approval submission: if payment is collected, route to "APPROVED" status; if no payment, then route to approval workflow.

**Data Mapping Gap:** Currently, Shopify integration may not be populating `Amount_Paid__c` on sync, or the field population is decoupled from the approval routing logic.
