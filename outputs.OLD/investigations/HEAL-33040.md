## Salesforce Problem

### Confirmed Root Cause

**DISABLED SCHEDULED FLOW — NOT A BUG IN SYNC LOGIC**

Flow: `Scheduled_Flow_Sync_Accounts_to_External_Systems`
- **Status:** `Obsolete` (DISABLED)
- **Purpose:** Run daily to sync missing accounts to external systems (Shopify, etc.)
- **Schedule:** Daily at 18:00:00 UTC
- **Active since:** 2023-11-10 (original schedule date)

**What it does:**
1. Queries Accounts by Record Type: `Patient_Account_Record_Type`
2. Filters for missing required fields (Gender, DOB, Billing Street, Email) and missing Biocanic_External_Id
3. Invokes subflow: `Autolaunch_Account_Sync_To_External_Systems`
4. Subflow contains Shopify Customer ID mapping and sync logic

**Why accounts not syncing:**
- Parent scheduled flow marked `Obsolete` = disabled
- Subflow `Autolaunch_Account_Sync_To_External_Systems` (Active) never invoked
- No accounts processed for Shopify sync
- Silent failure (no error messages generated)

### Impact Timeline
- 2026-04-10 onwards: 706+ accounts created but never synced
- Flow has been disabled for unknown duration (predates 2026-04-13 issue report)
- Still disabled as of 2026-05-22 (scope still growing daily)

### Solution

**URGENT: Subflow has critical bugs. Re-activating scheduled flow will NOT fix issue.**

Subflow `Autolaunch_Account_Sync_To_External_Systems` (ID: 301Rh00000MdqwZIAR) has three critical defects:

1. **eventId assigned NULL** — Set Event Details assigns NULL to eventId, passes broken data to Patient_Event_API
2. **API returns NO output** — Patient_Event_API returns zero data, can't capture Shopify customer ID
3. **No Account update step** — Flow never writes Shopify_Customer_ID__c back to Account even if API succeeds
4. **Silent failure** — No error handling, flow logs "success" even when sync fails

**Testing confirmed (2026-05-22):**
- Manual subflow invocation with account 0010b00002LQU4xAAH (Jason Toff)
- Flow completed without errors but returned no Shopify ID
- Account still has null Shopify_Customer_ID__c

**Fix required:**
1. Fix eventId assignment (remove NULL value)
2. Capture Patient_Event_API output
3. Add Account update step to persist Shopify customer ID
4. Add error handling with meaningful failure messages

**Before re-activating scheduled flow, subflow must be fixed in Engineering.**
