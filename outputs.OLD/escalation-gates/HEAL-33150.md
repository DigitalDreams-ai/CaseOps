# Escalation Gate - HEAL-33150

## Decision

Escalate to Engineering/Admin.

## Reason

The confirmed failure is in the email reply path for internal approval emails. Salesforce logged the outbound Cx Case email, but the corresponding Outlook reply was not delivered to Salesforce. A thread token was present, but Salesforce cannot attach a reply it never receives.

This is not supported as a generic missing Customer Experience Email-to-Case route. Production shows Customer Experience replies do attach when delivered to Salesforce-routed mailboxes such as `support@10xhealthsystem.com` or `info@10xhealthsystem.com`.

## Required Engineering/Admin Audit

1. Identify the Cx internal approval Send Email action/template/default sender.
2. Confirm the actual From and Reply-To values received in Outlook.
3. Determine whether a shared approval/refund mailbox receives these replies.
4. Configure the reply path so required replies are delivered to Salesforce through Email-to-Case or equivalent supported processing.
5. Validate in Sandbox before Production configuration changes.

## Gate Result

Support should not implement a blind Cx routing-address change. The next action is a targeted Engineering/Admin audit of Reply-To and mailbox forwarding.
