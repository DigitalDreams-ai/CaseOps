Salesforce CLI Authentication

Salesforce access tokens are automatically refreshed using OAuth 2.0 refresh tokens. No manual refresh needed.

Authenticated Organizations:

📋 Initial Setup (Required Once)

1️⃣ Authenticate Production Org
On your local machine, run:
sf org login web --alias 10xhealth
Logs in to production. Close browser after authorizing.

2️⃣ Authenticate Sandbox Org
Run for sandbox (requires test.salesforce.com URL):
sf org login web --alias 10xhealth-sean --instance-url https://test.salesforce.com
Logs in to sandbox. Close browser after authorizing.

3️⃣ Get Production Access Token
Run:
sf org auth show-access-token -o 10xhealth --json
Copy result.accessToken. This command shows the access token only.

4️⃣ Get Production Refresh Token
Run the command below and copy result.sfdxAuthUrl; CaseOps accepts that full URL in the refresh-token field.
sf org auth show-sfdx-auth-url -o 10xhealth --json


5️⃣ Get Sandbox Access Token
Run:
sf org auth show-access-token -o 10xhealth-sean --json
Copy result.accessToken.

6️⃣ Get Sandbox Refresh Token
Run the command below and copy result.sfdxAuthUrl; CaseOps accepts that full URL in the refresh-token field.
sf org auth show-sfdx-auth-url -o 10xhealth-sean --json


7️⃣ Save All Tokens
Visit Refresh Tokens page and paste all 4 tokens (Prod access, Prod refresh, Sandbox access, Sandbox refresh), then save.

✅ Automatic Refresh
Once setup, tokens are automatically refreshed on service startup if they're older than 4 hours. Access tokens valid for 8 hours; refresh happens before expiry. No manual action needed.

🔐 Token Storage
Production: Stored in SF_PROD_ACCESS_TOKEN with refresh token
Sandbox: Stored in SF_SANDBOX_ACCESS_TOKEN with refresh token
Both in .env.jira on the NAS.
